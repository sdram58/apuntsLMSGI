Variables en CSS

A CSS, les propietats personalitzades (també conegudes com a variables) són entitats definides per autors de CSS que contenen valors específics que es poden tornar a utilitzar en un document. S'estableixen mitjançant la notació de propietats personalitzades (per exemple, --main-color: black;) i s'hi accedeixen mitjançant la funció var() (per exemple, color: var (--main-color);).

Llocs web complexos tenen quantitats molt grans de CSS, sovint amb una gran quantitat de valors repetits. Per exemple, el mateix color es pot utilitzar a centenars de llocs diferents, la qual cosa requereix buscar i reemplaçar globalment si necessitem canviar aquest color. Les variables CSS permeten que un valor s'emmagatzemi en un lloc i després es faci referència a diversos altres llocs. Un benefici addicional són els identificadors semàntics. Per exemple --main-text-color és més fàcil d'entendre que #00ff00, especialment si aquest mateix color també es fa servir en un altre context.

Les variables CSS estan subjectes a la cascada, i hereten el valor del pare.

Ús Bàsic

Per declarar propietats personalitzades (variables) fem servir un nom que comenci amb dos guions (--), i un valor que pot ser qualsevol valor vàlid de CSS. Com qualsevol altra propietat, l'escrivim dins d'un set de regles d'estil, així:

element {
  --main-bg-color: brown;
}

Tingueu en compte que el selector que fem servir per a les regles d'estil defineix l'àmbit (scope) en què podrem fer servir la propietat personalitzada (variable). Una bona pràctica comuna és declarar variables a la pseudo-classe :root, i així aplicar-les globalment al document HTML:

:root {
  --main-bg-color: brown;
}

Tanmateix, això no ha de ser sempre així: podrien haver-hi molt bones raons per voler limitar l'àmbit de les teues propietats personalitzades.

Els noms de propietats personalitzades són case sensitive (distingeixen entre majúscules i minúscules) — --my-color serà tractat com una propietat personalitzada diferent de --My-color.

Com hem esmentat anteriorment, per accedir al valor d'una propietat personalitzada fem servir el nom de la propietat dins de la funció var(), en lloc de qualsevol altre valor normal:

element {
  background-color: var(--main-bg-color);
}

Herència de propietats personalitzades

Les propietats personalitzades hereten. Això vol dir que si no s'estableix cap valor per a una propietat personalitzada en un element donat, s'utilitza el valor del vostre element pare. Vegem el següent HTML:

<div class="un">
  <div class="dos">
    <div class="tres"></div>
    <div class="quatre"></div>
  </div>
</div>

amb el següent CSS:

.dos {
  --test: 10px;
}

.tres {
  --test: 2em;
}

En aquest cas, els resultats de var(--test) són:

  • Per a l'element class="dos": 10px
  • Per a l'element class="tres": 2em
  • Per a l'element class="quatre": 10px (heretat del pare)
  • Per a l'element class="un": valor no vàlid, que és el valor per defecte de qualsevol propietat personalitzada.

Tingueu en compte que aquestes són propietats personalitzades, no variables reals com podem trobar en altres llenguatges de programació. El valor es calcula on siga necessari, no s'emmagatzema per utilitzar-lo en altres regles. Per exemple, no es pot declarar una propietat per a un element i esperar recuperar-la a les regles del descendent d'un germà. La propietat només està configurada per al selector que coincidisca i els seus descendents, com qualsevol CSS normal.

Valors de substitució (fallback) de propietats personalitzades

Utilitzant var() podem definir múltiples valors de substitució (fallback) que es faran servir quan la variable donada no està definida encara; això pot ser útil quan es treballa amb Custom Elements i Shadow DOM

Els valors de substitució (fallback) no es fan servir per arreglar problemes de compatibilitat del navegador. Si el navegador no té suport per a Propietats Personalitzades de CSS, el valor de fallback no és d'ajuda. És simplement un suport perquè aquells navegadors que sí que suporten Propietats Personalitzades de CSS puguen triar un valor diferent en cas que la variable no s'haja definit o continga un valor no vàlid.

El primer argument a la funció és el nom de la propietat personalitzada que se substituirà. El segon argument a la funció, si es proporciona, és un valor de reserva, que s'utilitza com a valor de substitució (default) quan la propietat personalitzada referenciada no és vàlida. La funció només accepta dos paràmetres, assignant tot allò que després de la primera coma com el segon paràmetre. Si aquest segon paràmetre és invàlid, per exemple una llista de valors separats per comes, el fallback fallarà. Per exemple:

.dos {
  color: var(--my-var, red); /* Roig (red) si --my-var no està definida */
}

.tres {
  background-color: var(
    --my-var,
    var(--my-background, pink)
  ); /* Rosa (pink) si my-var i --my-background no estan definides */
}

.tres {
  background-color: var(
    --my-var,
    --my-background,
    pink
  ); /* Invàlid: "--background, pink" */
}

Com veiem en el segon exemple de dalt, la manera correcta d'incloure més d'un fallback és fer servir una propietat personalitzada com a fallback (la qual té el seu propi fallback). Aquesta tècnica s'ha vist que pot causar problemes de rendiment en prendre més temps en analitzar les variables.

La sintaxi del fallback, com la de les propietats personalitzades, permet comes. Per exemple, var(--foo, red, blue) defineix un fallback de red, blue — és a dir, qualsevol cosa entre la primera coma i el final de la funció es considera un valor de fallback.

Validesa i valors

El concepte clàssic de validesa en CSS, vinculat a cada propietat, no és molt útil pel que fa a les propietats personalitzades. Quan s'analitzen els valors de les propietats personalitzades, el navegador no sap on s'utilitzaran, per la qual cosa ha de considerar quasi tots els valors com a vàlids.

Malauradament, aquests valors vàlids es poden fer servir, a través de la notació funcional var(), en un context en què potser no tinguen sentit. Propietats i variables personalitzades poden portar a declaracions CSS no vàlides, donant lloc al nou concepte de vàlid en temps calculat.

Què passa amb els valors invàlids?

Quan el navegador troba una substitució invàlida amb var(), es farà servir el valor inicial o heretat. Considera el fragment següent:

<p>This paragraph is initial black.</p>

:root {
  --text-color: 16px;
}
p {
  color: blue;
}
p {
  color: var(--text-color);
}

Com esperaríem, el navegador substitueix var(--text-color) amb el valor de --text-color, però 16px no és un valor vàlid per a la propietat color. Després de la substitució, la propietat no té sentit. El navegador maneja aquesta situació en dos passos:

  1. Comprova si la propietat color és pot heretar. Sí, però l'element

    no té un pare amb una propietat de color. Així que passem al següent pas.

  2. Pren el valor del valor inicial per defecte.

Caixa flexible (flexbox)

display:flex

La propietat display permet una visualització d'elements coneguda com a flex, que permet distribuir els continguts en forma de caixa flexible. Es tracta de columnes adaptables, que admeten mides i alineacions variables.

És molt interessant i potent, encara que és una forma de maquetar prou moderna, quasi tots els navegadors actuals la permeten.

A la capa contenidora se li poden aplicar propietats que modifiquen aquest comportament per defecte. Totes només funcionen si s'ha indicat que tenim una distribució flexible (display:flex) a la capa contenidora.

La propietat flex-direction

Indica en quina adreça es distribueixen els continguts de la capa. Pot ser:

  • row. En forma de fila d'esquerra a dreta. Es el valor por defecto.
  • row-reverse. El fila, pero de derecha a izquierda
  • column. En forma de columna de arriba a abajo
  • column-reverse. En forma de columna de abajo a arriba


flex-wrap.

Per defecte, els continguts es distribueixen en una única línia (valor no-wrap). Aquesta propietat permet que es distribueixin en diverses línies si cal (valor wrap).


A la imatge anterior, les capes estan configurades amb les propietats flex-direction:row i flex-wrap:wrap, de manera que es distribueixen en forma de fila però ara es respecta la seva mida fent que ocupen les capes interiors dues files.

flex-flow

Permet assignar mitjançant una sola propietat les dues anteriors. Exemple

flex-flow:row wrap;

justify-content

Indiquen com es distribueixen les capes interiors dins de la capa contenidora. Només té efecte quan s'ha triat flux de tipus envoltant (wrap). Aquesta propietat té aquests valors:

  • flex-start. Les capes interiors es distribueixen alineades a l'esquerra
  • flex-end. Les capes interiors es distribueixen alineades a la dreta
  • center. Les capes interiors apareixen centrades a la capa contenedora
  • space-between. Les capes interiors apareixen distribuïdes de manera justificada a l'esquerra i la dreta deixen espais entre elles per aconseguir aquest efecte.
  • space-around. Semblant a l'anterior, només que en aquest cas l'espai es deixa entre les capes i també per fora.

Gràficament (usant les capes anteriors com a exemple) aquestes són totes les possibilitats:


align-items

En aquest cas, s'indica com es distribueixen les capes interiors al llarg de l'eix vertical. Possibilitats:

  • flex-start. Les capes interiors es distribueixen de dalt a baix.
  • flex-end. Les capes interiors s'alineen a la vora inferior.
  • center. Les capes interiors se centren en vertical
  • stretch. Les capes estiren la mida perquè ocupin des de la vora superior fins a la vora inferior.
  • baseline. Les capes interiors es distribueixen des de la línia base del contingut (sense contingut, l'alineació no té efecte).


Aquestes són les alineacions aconseguides usant l'exemple de capes vist. Per al darrer resultat (stretch) s'ha retirat l'alçada (propietat height) de les capes interiors; altrament stretch no podria aconseguir aquest resultat.


align-content

Propietat similar a justifiy-content, però referida a l'eix vertical. Té més possibilitats que align-item, per la qual cosa es fa servir més habitualment que ella. En realitat són diferents, els seus resultats difereixen prou. Possibles valors:

  • flex-start. Les capes interiors s'apilen de dalt a baix.
  • flex-end. Les capes interiors s'apilen a la vora inferior.
  • center. Les capes interiors s'apilen al centre del contenidor.
  • stretch. Les capes estiren la mida perquè ocupin des de la vora superior fins a la vora inferior.
  • space-between. Les capes s'alineen de dalt a baix i deixen un espai proporcional entre cada línia.
  • space-around. Les capes s'alineen de dalt a baix i deixen un espai proporcional entre cada línia. A més, aquest mateix espai entre cada línia es deixa a dalt de la primera línia i sota l'última, per la qual cosa no queden situades just a les vores.

Pàgines Adaptables

El disseny web adaptable o adaptatiu (en anglès, Responsive Web Design) és una filosofia de disseny i desenvolupament web que, mitjançant l'ús d'estructures i imatges fluides, així com de @media queries al full d'estil CSS, aconsegueix adaptar el lloc web a l'entorn del usuari.

El dissenyador i autor nord-americà Ethan Marcotte va crear i difondre aquesta tècnica a partir d'una sèrie d'articles a "A List Apart" (http://alistapart.com/article/responsive-web-design/), una publicació en línia especialitzada en disseny i desenvolupament web, idea que després estendria al seu llibre Responsive Web Design.

Origen

Tant la idea com el propòsit del disseny web adaptable van ser prèviament discutits i descrits pel consorci W3C al juliol de 2008 en la seva recomanació "Mobile Web Best Practices" sota el subtítol "One Web".

El concepte de "One ​​Web" fa referència a la idea de construir una Web per a tots (Web for All) i accessible des de qualsevol tipus de dispositiu (Web on Everything).

Aquesta recomanació, encara que específica per a dispositius mòbils, puntualitza que està feta a el context de "One Web", i que, per tant, no només engloba l'experiència de navegació a dispositius mòbils, sinó també en dispositius de més resolució de pantalla com dispositius de sobretaula.

Avui dia, la varietat de dispositius existents al mercat ha provocat que la informació disponible no siga accessible des de tots els dispositius, o bé és accessible, però l'experiència de navegació és molt pobre.

Avantatges

El principal avantatge que trobem amb el disseny web adaptable és que el web es visualitzarà correctament a tots els dispositius que usem i s'adaptarà als girs en dispositius mòbils.

A més, Google té en compte les pàgines web que tenen dissenys sensibles o adaptables, gràcies al seu Googlebot-Mobile. El personal de Google recomana que utilitzeu el Responsive Design per crear pàgines web que s'adapten a tots els dispositius utilitzant @media-queries. La alternativa seria utilitzar diferent codi segons l'agent del dispositiu, o fins i tot, diferents URLs. Google ens incita a fer servir aquest mètode perquè, d'aquesta manera, Googlebot no necessita analitzar tant contingut de la mateixa web, cosa que us facilita la tasca d'assignar el tipus de contingut de la pàgina i daquesta manera millorarà el SEO de la web.

“Google can discover your content more efficiently as we wouldn’t need to crawl a page with the different Googlebot user agents to retrieve and index all the content".

Un altre factor, que normalment no tenim en compte a l'hora de fer un disseny, és la accessibilitat del mateix, que dificulta algunes vegades l'ús duna web a un disminuït visual. Amb un disseny adaptable que aprofiti lespai de la pàgina podem oferir una pàgina de qualitat per a aquest col·lectiu.

A continuació, veurem allò essencial per crear un disseny web adaptable: el meta-tag Viewport, les @media queries, etc.

Viewport

Aquesta meta-etiqueta va ser creada en principi per Apple per al mòbil predilecte, però s'ha convertit en tot un estàndard que és suportat per la majoria dels dispositius mòbils (smartphones, tablets i gran part de mòbils de gamma mitjana i baixa).

El seu ús és totalment necessari, ja que sinó el navegador estableix l'amplada amb què prefereix visualitzar una pàgina, en lloc d'usar l'amplada de què disposa, és a dir, si la pantalla de el nostre mòbil té 400px i el navegador detecta que allò òptim seria visualitzar-la amb 700px, així ho farà, llevat que usem aquesta meta.

Com sempre, el meta l'afegirem al <head> de la nostra pàgina:

<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no">

Es poden utilitzar els paràmetres següents (separats per comes):

  • width: ample de la pàgina (es pot establir en píxels o com a device-width i utilitzarà l'amplada de què disposa el dispositiu).
  • height: alt de la pàgina, actua igual que el width.
  • initial-scale: escala o zoom inicial de la pàgina (aquest i els altres tipus d'escala s'estableixen amb valors com 1.0 per no tenir zoom o 2.5 per tenir un zoom de 2 augments i mig, per exemple).
  • minimum-scale: zoom mínim que podem fer a la pàgina.
  • maximum-scale: zoom màxim que podem fer a la pàgina.
  • user-scalable: estableix si està permès o no fer zoom (yes/no).

Preparar la nostra pàgina per al disseny adaptable

Per aconseguir que la nostra web s'adapti a les amplades de pantalla, hem de tenir en compte diverses coses importants:

  • La primera, i la més important, és deixar d'usar píxels a tot hora, al seu lloc, farem servir percentatges (per exemple: width: 60%). Per saber quins percentatges devem de posar, farem la següent operació: Suposem que tenim un element amb un amplada de 960 píxels i, dins d'aquest, tenim un altre element de 300 píxels. A l'element extern li posarem, per exemple, un width del 90% perquè s'adapte a la mida de la finestra i perquè l'element intern mantinga la proporció respecte a l'extern calcularem l'amplada que hauria de tenir així: 300/960, cosa que ens donarà 0,3125. Per això tant, el width que posarem a l'element intern serà 31,25%.
  • Per limitar l'amplada (o alt si s'escau) hem d'usar el paràmetre max-width (maxheight en el cas de l'alt). Per establir el mínim faríem servir min-width i min-height, encara que aquests sutilitzen menys. El valor d'aquestes propietats sobreescriurà el valor de les propietats width i height. Per exemple, si indiquem width: 100% i max-width: 600px, quan visualitzem la pàgina amb una resolució de 1200px, l'amplada serà de 600px, ja que és el valor màxim que hem indicat per a l'amplada. En aquest exemple, tenim quatre elements configurats amb diferents combinacions de width i max-width. Si anem canviant l'amplada de la finestra del navegador, veurem com les amplades se'n van adaptant fins arribar a les amplades màximes configurades, moment en el qual, l'amplada romandrà invariable. A la següent imatge podem observar com es mostra el exemple.
  • No hem d'usar posicions absolutes ni fixes (llevat de comptades excepcions).
  • Mai no hem de permetre que una imatge de fons que està pensada per no repetir-se, arribi a repetir-se pel canvi de dimensions. Per evitar-ho, l'hem d'adaptar, normalment, usant @media-queries.
  • Tampoc no hem de permetre que les imatges i els vídeos isquen de l'estructura, sinó apareixerà un scroll lateral als dispositius mòbils que destrossarà totalment el disseny.

@media queries

Des dels dies de CSS 2.1, els nostres fulls d'estils han gaudit "d'una consciència del dispositiu" a través dels media types. Per exemple, quan creem un full d'estils per impressió, estem utilitzant aquest concepte:

<link rel="stylesheet" type="text/css" href="core.css" media="screen" />
<link rel="stylesheet" type="text/css" href="print.css" media="print" />

Per sort, el W3C va crear les media queries com a part de l'especificació CSS3, millorant la promesa dels media types. Una media query ens permet apuntar, no només a certes classes de dispositius, sinó realment inspeccionar les característiques físiques del dispositiu que està renderitzant la nostra feina. Per exemple, seguint el recent creixement de WebKit mobile, les mitja volies s'han convertit en una popular tècnica del costat del client per lliurar una full destils a mida per a l'iPhone, els smartphones Android o els tablets. Per fer-ho, podem incorporar una query a l'atribut mitjà d'un full d'estils enllaçat:

<link rel="stylesheet" type="text/css" media="screen and (max-device-width: 480px)" href="shetland.css" />

La query conté dos components: Un media type (screen), i la consulta entre parèntesis, que conté una característica a inspeccionar (max-device-width) seguida pel valor al qual apuntem (480px). En altres paraules, us estem preguntant al dispositiu, si la seva resolució horitzontal (max-device-width) és igual o menor que 480px. En el cas que visualitzem la pàgina en un dispositiu amb una pantalla petita com l'iPhone, llavors el dispositiu carregarà shetland.css. Si no, el link s'ignora.

En el passat, aquest tipus de coses necessitaven solucions javascript, però l'especificació de media query proveeix una sèrie de característiques del medi que s'estenen molt més enllà de la resolució de la pantalla, ampliant l'abast del que podem testejar amb les nostres queries. A més, podem testejar múltiples valors de les propietats en una sola query, encadenant-los amb la paraula clau and:

<link rel="stylesheet" type="text/css" media="screen and (max-device-width: 480px) and (resolution: 163dpi)" href="shetland.css" />

A més de l'ús de l'etiqueta <link> per incloure media-queries, també tenim la possibilitat de incloure-les al nostre CSS com a part d'una regla @media:

@media all and (max-device-width: 480px) {
  .column {
    float: none;
  }
}

O com a part d'una directiva @import:

@import url("shetland.css") all and (max-device-width: 480px);

En cada cas, l'efecte és el mateix: si el dispositiu passa el test plantejat per la nostra mitjana query, el CSS que corresponga és aplicat al nostre codi. La paraula all indica que la regla serà per a tots els tipus de mitjans i no per a un de específic.

Amb això disposem de noves capacitats que ens permeten definir conjunts d'estils depenent de propietats comunes dels dispositius que accedeixen als nostres llocs. Propietats com l'alçada i l'amplada, la relació d'aspecte o el nombre de colors disponibles. Les regles @media poden ser utilitzades per adaptar les nostres pàgines, no només per a dispositius comuns, sinó per a tot tipus de dispositius que els nostres lectors usin per visitar-les. Tot i que les @media queries ens permet conèixer moltes propietats diferents, les més importants són les següents:

  • aspect-ratio: Fa referència a les dimensions relatives del dispositiu expressades com una relació daspecte: 16:9 per exemple.
  • width i height: fa referència a les dimensions de l'àrea de visualització. A més, poden ser expressades en valors mínims i màxims.
  • orientation: Detecta si el layout és panoràmic (l'amplada és més gran que l'alt) o vertical (el alt és més gran que l'amplada). Això ens permet ajustar els dissenys per a dispositius amb propietats de gir de la pantalla com l'iPhone i altres smartphones o les tauletes.
  • resolution: Indica la densitat dels píxels al dispositiu de sortida. Això és especialment útil quan volem aprofitar-nos dels avantatges dels dispositius que té una resolució més gran de 72 dpi.
  • color, color-index i monochrome: Detecten el nombre de colors o bits per color. Això ens permet crear dissenys específics per a dispositius monocromàtics.

Posada en marxa

Ja hem vist el bàsic que necessitem conèixer sobre el Responsive Design, però, la majoria de de vegades, el més difícil és aplicar-lo de la manera que necessitem o mostrar el contingut d'una forma eficaç sense perdre en disseny ni usabilitat, i en això és el que ens centrarem ara, a estructurar correctament el contingut duna pàgina que sadapti a diferents tipus de pantalla.

Per començar veurem com fer fluir elements com imatges, blocs de text o qualsevol cosa en general, de manera que es reubiquin els elements en funció de la mida disponible a la pantalla. En primer lloc, ens oblidarem dels float i farem servir en el seu lloc la propietat display.

Com ja sabem, la propietat display de CSS ens permet establir com es comportarà un element respecte a els altres. Ens interessen, principalment, tres valors possibles:

  • block: els elements s'ubiquen un baix l'altre.
  • inline: els elements s'ubiquen un al costat de l'altre, però sense respectar les propietats de marge.
  • inline-block: aquesta última forma de visualitzar-se un element és una barreja de les dues anteriors, comportant-se com un bloc, però posant-se en línia amb altres elements iguals.

Una de les coses que es donen molt en els dissenys adaptatius és crear un conjunt de blocs amb imatges i text que, segons la mida de la finestra, es van recol·locant. Per exemple, a la web de la imatge es veuen sis blocs, tres a sobre i tres a sota, però si els visualitzéssim amb una amplada de pantalla més gran es veurien en línia.


Vegem com podríem crear una estructura com aquesta:

<div id="contenidor">
  <figure class="bloc">[Text/Imatges]</figure>
  <figure class="bloc">[Text/Imatges]</figure>
  <figure class="bloc">[Text/Imatges]</figure>
  <figure class="bloc">[Text/Imatges]</figure>
  <figure class="bloc">[Text/Imatges]</figure>
  <figure class="bloc">[Text/Imatges]</figure>
</div>

Al full d'estils, simplement, establirem el mode de visualització display:inline-block per a l'element figure, i quan els elements no càpiguen al contenidor es reajustaran sense perdre la mida.

#contenidor .bloc{
  display: inline-block;
  height:300px;
  width: 300px;
  border:1px solid #333;
  background: #999;
  margin:20px;
}


Si proveu a ampliar i reduir la mida de la finestra del vostre navegador, veureu que els elements es van reubicant automàticament gràcies a la propietat display.

Molt bé, però què passa si volem que els elements es comporten d'una manera diferent quan la mida de finestra disponible siga inferior a un valor determinat? En aquest cas, utilitzarem les media-queries. Per exemple, si volguéssim que a partir dels 800 píxels els blocs anteriors es mostraran uns sota els altres, centrats i ajustant la seva amplada a l'espai disponible, farem el següent:

#contenidor .bloc{
  display: inline-block;
  height:300px;
  width: 300px;
  border:1px solid #333;
  background: #999;
  margin:20px;
}

@media all and (max-width: 800px){
  #contenidor .bloc{
    /* Quan l'amplada siga inferior a 800px l'element serà un bloc */
    display: block;
    width: auto;
  }
}

Nota: Com que el selector és el mateix (display i block), hem de posar la media-query baix, així serà aquesta la que s'aplique. Si la posarem d'alt no faria res perquè la de baix la sobreescriurà. Ho podriem solucionar amb !important, pero ja sabeu que està desaconsellat.

Passar de columnes a cascada

Un dels mètodes que s'utilitza més sovint per canviar l'estructura d'una web adaptativa, que sol tindre una capçalera i dues o tres columnes (amb el menú, el contingut, enllaços, etc.) és el pas de la disposició en columnes a la disposició a cascada.


A la imatge es pot veure com, en funció de l'amplada disponible es canvia la disposició.

A continuació, veurem com fer-ne una estructura adaptable similar a la de la imatge, que estaria formada per un títol amb un menú, una capçalera i dues columnes, la primera amb una agenda amb esdeveniments i la segona columna amb els continguts.

El que farem perquè siga totalment adaptatiu és fer xicotets canvis segons les necessitats, però per a l'exemple només tindrem dos estats: l'estructura ben maquetada i l'estructura en cascada, com es representa a la imatge anterior.

El nostre html serà el següent:

<nav>[Menú]</nav>
<div class="pagina">
  <header>[Capçalera]</header>
  <aside>[Agenda]</aside>
  <main>[Contingut]</main>
</div>

I aplicarem els estils següents:

nav{
  background: #F99;
  position: fixed;
  top: 0px;
  left: 0px;
  width: 100%;
  height: 30px;
}
.pagina{
  /* la pàgina serà com a màxim de 1000px d'amplada */
  max-width: 1000px;
  margin:auto;
}
header{
  background: #9F9;
  margin-top: 30px;
  height: 200px;
}
aside{
  background: #99F;
  float: left; /* surem l'element a l'esquerra */
  width: 35%;
  height: 600px;
}
main{
  background: #AAA;
  width: 65%;
  margin-left: 35%;
  height: 900px;
}

@media all and (max-width: 600px){
  div, nav, header, aside, main{
    /* Quan l'amplada sigui inferior a 600px l'element serà un bloc */
    display: block;
    /* S'ajustaran a l'amplada de la finestra */
    width: 100%;
    margin: auto;
    /* La posició serà estàtica (flux normal del document) */
    position: static;
    /* els elements deixaran de ser flotants */
    float: none;
  }
}

Per provar l'exemple, simplement, heu de carregar la pàgina al navegador i reduir-la finestra fins que tinga una resolució inferior a 600 píxels.


Adaptar imatges i vídeos

Moltes vegades, per crear dissenys molt optimitzats pot ser important fer servir diverses imatges diferents, una amb una mida superior i una altra amb una de més xicoteta, però ens centrarem en el ús d'una mateixa imatge que, en la mesura del possible, hauria d'estar optimitzada i adaptarà com nosaltres vulguem.

Podem fer que una imatge es comporte de les següents formes:

  • Ocupar l'amplada de la pàgina (o contenedor). Pot ser útil per a capçaleres o imatges principals i ho farem establint l'amplada al 100%. 
    img { width:100%; }

  • Tenir una imatge amb una mida màxima. En aquest cas, la imatge tindrà una mida que no sobrepassarà, però quan es disminuïsca la mida del contenedor s'encongeix adaptant-se a la pàgina. 
    img { width:100%; màx-width:400px; }

  • Tenir les imatges amb la seva mida original com a màxim. Aquest cas és molt similar al anterior, amb la diferència que posarem com a mida màxima l'amplada de la imatge. De aquesta forma, la imatge no es deformarà en ser redimensionada a una mida superior a la original. 
    img { max-width:100%; }

  • Per adaptar la mida dels vídeos farem servir les mateixes tècniques que utilitzem per a les imatges.

Una cosa a tenir en compte, és que si hem posat un alt fix a un vídeo, i després, en una @media query, posem, per exemple, una amplada relativa utilitzant percentatges, l'element <video> es redimensionarà per mantenir les proporcions en redimensionar la finestra, però el espai reservat per a l'alt inicial es mantindrà reservat, de manera que els elements que es troben al voltant no es reubicaran. Per solucionar aquest problema, n'hi haurà prou amb indicar a la @media query que l'alt serà automàtic (height:auto).

Grid View

Què és una Grid-View?

Moltes pàgines estan basades en una grid-view, això significa que la pàgina està dividida en columnes:


Utilitzar una grid-view és molt útil quan es dissenyen llocs web. Això ens facilita la reubicació dels elements a la pàgina.

Una grid-view responsiva ofereix 12 columnes, amb una amplada total del 100%, i es reduirà i expandirà a mesura que canvieu la mida de la finestra del navegador


Construir una Grid-View Responsiva

Començarem a construir GridView responsiva.

Primer assegureu-vos que tots els elements HTML tinguen la propietat de mida de quadre establerta en quadre de vora. Això assegura que el reompliment i la vora s'incloguen a l'amplada i l'alçada totals dels elements.

Afegiu el codi següent al CSS:

* {
  box-sizing: border-box;
}

El següent exemple mostra un senzill exemple de web responsiva amb dues columnes:


.menu {
  width: 25%;
  float: left;
}

.main {
  width: 75%;
  float: left;
}

L'exemple anterior està bé si la pàgina web només conté dues columnes.

No obstant això, farem servir una vista de quadrícula amb 12 columns per tindre més control sobre la pàgina web.

Primer cal calcular el percentatge per a una columna: 100% / 12 columns = 8.33%.

Després fem una classe per a cada una de les 12 columnes, class = "col-" i un nom que defineix quantes columns ocuparà la secció:

.col-1 {width: 8.33%;}
.col-2 {width: 16.66%;}
.col-3 {width: 25%;}
.col-4 {width: 33.33%;}
.col-5 {width: 41.66%;}
.col-6 {width: 50%;}
.col-7 {width: 58.33%;}
.col-8 {width: 66.66%;}
.col-9 {width: 75%;}
.col-10 {width: 83.33%;}
.col-11 {width: 91.66%;}
.col-12 {width: 100%;}

Totes les columnes han de surar a l'esquerra i tenen un padding de 15px.

[class*="col-"] {
  float: left;
  padding: 15px;
  border: 1px solid red;
}

Cada fila ha d'estar dins d'un <div>. El número de columnes dins una fila sempre ha de sumar fins a 12:

Exemple:

<div class="row">
  <div class="col-3">...</div> <!-- 25% -->
  <div class="col-9">...</div> <!-- 75% -->
</div>

Les columnes dins d'una fila (row) estan flotant cap a l'esquerra i, per tant, s'eliminen del flux de la pàgina, i altres elements es col·locaran com si les columnes no existissin. Per evitar això, afegirem un estil que esborre el flux:

.row::after {
  content: "";
  clear: both;
  display: table;
}

Afegir punts de ruptura

Hem creat una pàgina web amb files i columnes, però no es veia bé en una petita pantalla.

Podem afegir un punt d'interrupció on certes parts del disseny es comportaran de manera diferent a cada costat del punt dinterrupció.


Use una media query para agregar un punto de interrupción a 768px:

/* For desktop: */
.col-1 {width: 8.33%;}
.col-2 {width: 16.66%;}
.col-3 {width: 25%;}
.col-4 {width: 33.33%;}
.col-5 {width: 41.66%;}
.col-6 {width: 50%;}
.col-7 {width: 58.33%;}
.col-8 {width: 66.66%;}
.col-9 {width: 75%;}
.col-10 {width: 83.33%;}
.col-11 {width: 91.66%;}
.col-12 {width: 100%;}

@media only screen and (max-width: 768px) {
  /* For mobile phones: */
  [class*="col-"] {
    width: 100%;
  }
}

Quan la pantalla (finestra del navegador) és més estreta que 768px, cada columna ha de tindre una amplada del 100%.

Això significa que hem de fer alguns canvis al nostre CSS.

En lloc de canviar els estils quan l'amplada és menor que 768px, hauríem de canviar el disseny quan l'amplada siga més gran que 768px. Això farà que el nostre disseny Mobile First:

  .col-7 {width: 58.33%;}
  .col-8 {width: 66.66%;}
  .col-9 {width: 75%;}
  .col-10 {width: 83.33%;}
  .col-11 {width: 91.66%;}
  .col-12 {width: 100%;}
}

Pot semblar estrany que tinguem dos conjunts de classes idèntiques, però ens dóna l'oportunitat a HTML, per decidir què passarà amb les columnes a cada punt d'interrupció:

Per escriptori: (col-)

La primera i la tercera secció ocuparan 3 columnes cadascuna. La secció central inclourà 6 columnes.

Per a tauletes: (col-s-)

La primera secció inclourà 3 columnes, la segona ocuparà 9, i la tercera secció es mostrarà sota de les dues primeres seccions, i ocuparà 12 columnes:

<div class="row">
  <div class="col-3 col-s-3">...</div>
  <div class="col-6 col-s-9">...</div>
  <div class="col-3 col-s-12">...</div>
</div>

Punts típics d'interrupció del dispositiu

Hi ha milers de pantalles i dispositius amb diferents alçades i amples, per la qual cosa és difícil crear un punt de tall exacte per a cada dispositiu. Per mantindre les coses simples, podriem apuntar cinc grups:

/* Extra small devices (phones, 600px and down) */
@media only screen and (max-width: 600px) {...}

/* Small devices (portrait tablets and large phones, 600px and up) */
@media only screen and (min-width: 600px) {...}

/* Medium devices (landscape tablets, 768px and up) */
@media only screen and (min-width: 768px) {...}

/* Large devices (laptops/desktops, 992px and up) */
@media only screen and (min-width: 992px) {...}

/* Extra large devices (large portàtils and desktops, 1200px and up) */
@media only screen and (min-width: 1200px) {...}

Orientació: Portrait / Landscape

Les queries també es poden utilitzar per canviar el disseny d'una pàgina segons l'orientació del navegador.

Podeu tenir un conjunt de propietats CSS que només s'aplicaran quan la finestra del navegador siga més ampla que la seva altura, el que s'anomena orientació "Landscape":

Exemple:
En aquest exemple la pàgina tindrà color de fons lightblue si l'orientació és landscape:

@media only screen and (orientation: landscape) {
  body {
    background-color: lightblue;
  }
}

Oculta Elements amb Media Queries

Un altre dels usos comuns de les media queries, és amagar elements en diferents mides de pantalla:

Si vull amagar informació en pantalla xicotetes per exemple es pot fer algo així:

/* If the screen size is 600px wide or less, hide the element */
@media only screen and (max-width: 600px) {
  div.example {
    display: none;
  }
}
