El llenguatge HTML

Introducció

A finals dels anys 80 es va desenvolupar el llenguatge de marques SGML. Al final d'aquesta dècada Tim Bernes Lee, científic britànic que treballava al CERN, va utilitzar SGML per definir un nou llenguatge d'etiquetes que va anomenar Hypertext Markup Language (llenguatge de marcat d'hipertext) per crear documents transportables a través d'Internet on fos possible l'hipertext.

S'anomena hipertext a la possibilitat que en un document determinades paraules marcades de manera especial permetin obrir un document relacionat amb elles; alguna cosa a què ara estem molt acostumats, però que abans era molt innovador.

A més d'HTML, l'equip de Tim Bernes Lee va definir les bases del protocol http d'Internet, encarregat de transportar els documents HTML i els recursos associats. La versió definitiva d'http es va fer l'any 1999 per part de l'organisme World Wide Web Consortium (W3C).

Tot i tardar a ser acceptat, HTML va ser un èxit rotund i la causa indubtable de l'èxit dInternet. Avui dia, quasi tot a Internet es veu a través de documents HTML que, popularment, s'anomenen pàgines web.

Inicialment aquests documents es veien amb ajuda d'intèrprets de text (com l'antigament famós Lynx d'Unix) que reconeixien el codi HTML però que tenien capacitat només de pintar el text especial (com les negretes i cursives per exemple) i remarcaven l'hipertext, sobre el qual la tecla tabulador permetia accedir de forma seqüencial a tots els hipervincles del document.

Després el programari va millorar i van aparèixer ja els anomenats navegadors amb capacitat d'interpretar el codi HTML de manera més visual i canviar el tipus de lletra, colors, incorporar imatges, so, etc. És a dir, realment els documents HTML es van convertir en pàgines web.

Lògicament des del 1989 fins als nostres dies HTML ha millorat. Entre els seus avanços fonamentals:

  • El llenguatge cada cop ha anat incorporant noves etiquetes més potents, que permeten incloure als documents HTML, taules, capes, marcs, imatges,…
  • S'han afegit llenguatges de script (especialment JavaScript) amb codi incrustat a les pròpies pàgines HTML que permeten afegir funcionalitats extra i dinamisme a les pàgines web
  • Has aparegut llenguatges i tecnologies que permeten executar accions avançades al servidor que allotja les pàgines web. Entre elles CGI, PHP, ASP o JSP.
  • S'han incorporat llenguatges d'estil (com CSS) per a generar un format de document més avançat i fàcil de mantenir.
  • S'han afegit utilitats per a gestió avançada de JavaScript amb XML (AJAX) per donar encara més interactivitat i dinamisme a les pàgines.
  • S'ha permès la inclusió d'elements avançats a les pàgines com Flash (obsolet) o els applets de Java per donar més funcionalitat.
  • Es permeten elements semàntics per donar significat al contingut.
  • S'afegeixen cada cop més plugins als navegadors que així són capaços de mostrar imatges, so, vídeo i altres elements multimèdia a les pròpies pàgines.
  • Els llenguatges relacionats amb les pàgines web segueixen creixent i possibiliten afegir-ho tot tipus d'elements a les pàgines web, dibuixos, animacions, ús de localització mitjançant GPS,…

Funcionament de les aplicacions web

HTML, CSS i JavaScript

Una aplicació web, és una aplicació creada usant com a base el llenguatge HTML. Per tant, és una aplicació que s'executa en un navegador d'Internet.

Les aplicacions web utilitzen llenguatges que donen suport a la tasca d'HTML i que són traduïbles per part dels navegadors. Essencialment les aplicacions web actuals utilitzen:

  • HTML. Per donar significat als continguts de laplicació web. Permet indicar quins textos són títols, quins són paràgrafs normals, quines són cel·les duna taula, quines són imatges, etc.
  • CSS. Llenguatge que permet donar format i maquetació als continguts. Color, mida de lletra, posició, etc.
  • JavaScript. Permet dissenyar la interactivitat de la pàgina. Permet que les accions de l'usuari es puguin capturar i que la pàgina hi reaccione. Exemples de les possibilitats que quan l'usuari agafi el ratolí a una imatge aquesta s'il·lumine, o que quan l'usuari faci clic en un títol es mostre el contingut relacionat si estava ocult o que l'usuari puga augmentar la mida de la lletra de la pàgina fent clic a una icona.

Protocol http

La transmissió de pàgines web (que en definitiva són documents HTML) es fa mitjançant el protocol http, que és part de la pila de protocols TCP/IP. És un protocol basat en una comunicació petició-resposta; de manera que un client (també anomenat user agent, agent d'usuari) fa una petició de recurs indicant la seva adreça a Internet. La petició arribarà a un servidor http (també anomenat servidor web), el qual respon a aquesta petició, bé transmetent al client el recurs sol·licitat o bé indicant un missatge derror si el recurs no està disponible.


Els paquets http que s'intercanvien entre el client i el servidor contenen una capçalera amb informació de control i després el cos. Aquest cos és el codi HTML en el cas duna pàgina web, o la imatge, àudio, etc. que s'està transmetent.


A l'imatge anterior, s'observa el paquet retornat pel servidor web quan rep una petició http (de tipus GET) a la URL http://jorgesanchez.net. S'indica que la resposta va ser correcta (codi 200, OK), la IP del servidor que respon, la mida del paquet, la data en què expira, etc. A més s'annexa la capçalera de la petició original (request headers) on es veu el navegador de l'usuari que va fer la petició, el llenguatge del seu equip, etc.

A més de la capçalera, el contingut dels paquets http té un cos amb el contingut del recurs que se sol·licitava. Per exemple, en cas d'haver sol·licitat una pàgina web, el cos conté codi HTML que el navegador podrà traduir.

El protocol http serveix per transportar tota mena de continguts. De fet, el codi HTML d'una pàgina web fa referència a altres fitxers (imatges, fitxers CSS, etc.). Quan el navegador detecta aquestes referències, sol·licita al servidor web aquests recursos. El navegador respondrà amb nous paquets que contindran (si està a la mà) el contingut sol·licitat.

El client (normalment un navegador) serà l'encarregat de mostrar el recurs sempre que el navegador tinga la capacitat de fer-ho, ja que els navegadors poden mostrar molts formats de contingut, però per descomptat no tots.

Entre els recursos transmesos per http que un navegador és capaç de reconèixer estan:

  • Codi HTML, CSS i JavaScript
  • Imatges JPEG, PNG, SVG i GIF
  • Vídeos MP4 i altres formats
  • Àudio MP3 i altres formats
  • Fitxers XML (encara que no la validació ni el codi XSL associat)

Front-end i back-end

En aquest apartat s'explica breument el funcionament d'una aplicació web complexa. L'únic propòsit és indicar que les aplicacions web utilitzen nombroses tecnologies.

A la majoria d'aplicacions web complexes, quan un navegador requereix aquesta aplicació, el servidor necessita executar accions com: interpretar codi del costat del servidor (per exemple en PHP o ASP.Net), demanar recursos a altres servidors (bases de dades, mapes, streaming de vídeo, etc.) o emmagatzemar dades de sessió.

Aquestes accions són totalment opaques a l'usuari. Després de la seva execució, el navegador de l'usuari rep codi que sí que és capaç d'interpretar (normalment HTML, CSS i JavaScript).

Això implica que una aplicació web pot realitzar tasques a l'anomenat costat del servidor. És a dir, tasques que s'executen al servidor i el codi del qual no podem obtenir. Per tant, quan es crea una aplicació web hi ha persones dedicades a programar en codi traduïble al costat del servidor i codi al costat del client,

El costat del servidor s'associa al desenvolupament back-end, els seus programadors són anomenats programadors back-end. Mentre que els professionals encarregats de l'aparença al navegador, són anomenats programadors front-end.


HTML és la tecnologia dominant per programar el front-end. Per tant, aquest és un "manual" dedicat al front-end.

Eines per crear aplicacions en HTML

Per escriure codi HTML n'hi hauria prou amb un editor de text pla com el bloc de notes de Windows o vi veure de Linux/Unix. No obstant això, l'escriptura d'HTML en aquest tipus d'editors és incòmoda ja que no proporcionen ajuda ni acceleren l'escriptura.

A més, els resultats s'han de mostrar en un navegador o, encara millor, provar en diversos navegadors per comprovar problemes de compatibilitat (ja que hi ha elements HTML que no són compatibles amb tots els navegadors).

Així, la llista de navegadors a instal·lar per provar el nostre codi seria: Microsoft Internet Explorer, Google Chrome, Mozilla Firefox, Apple Safari i Opera. D'altra banda, és interessant provar en dispositius mòbils o bé afegir plugins als nostres navegadors que simulin la visualització d'un mòbil.

Per escriure el codi l'ideal és treballar mitjançant editors de codi capaços d'entendre el llenguatge i pintar de manera diferent les etiquetes HTML per distingir-les del text normal i així treballar millor i que a més incorporin eines que facilitin l'edició.

Així tenim les opcions següents:

  • Editors de text multipropòsit (editors de codi). Són programes que permeten editar text, però que estan pensats per escriure codi a qualsevol llenguatge de programació. El seu avantatge és que acoloreixen la sintaxi depenent del llenguatge en què estem escrivint, perquè són editors políglotes.
    Quan escrivim en HTML, acoloreixen el text ressaltant el codi de manera que sigui més llegible. A més ens ajuden a escriure codi HTML de forma més eficaç, perquè fins i tot detecten les fallades. Entre els més coneguts hi ha Notepad++, TextMate, Atom, Visual Studio Code o Sublim Text.
  • Editors de text HTML. Són editors especialitzats a escriure codi HTML. La diferència respecte als anteriors és que, amb aquests, no podrem escriure codi en llenguatges com C o Java. A canvi, tindrem més funcions per escriure en HTML. Per exemple, podrem previsualitzar el resultat del codi HTML (avançant-nos al navegador), pintat més ajustat als llenguatges de la web, reconeixement de llibreries i plantilles d'ús habitual de dissenyadors,…
    Alguns són: Coda, Komodo Edit, WebStorm o Brackets. Els editors Atom i Visual Studio Code n'hi ha que els col·loquen en aquest tipus perquè, encara que són editors políglotes, estan molt especialitzats en aplicacions web.
  • Editors XML. Pensats per a la gent que treballa creant documents i elements relacionats amb XML. Entenen HTML, però no són tan potents com els anteriors, encara que poden servir per a feines que no requereixen molta especialització en HTML. Els editors XML més famosos són Oxygen i XMLSpy.
  • Entorns complets de programació. Són els anomenats IDE (Entorns de Desenvolupament Integrat) que són entorns de treball per programar en llenguatges avançats com Java o C++, però que solen treballar perfectament per crear projectes per a la web. Destaquen Eclipse, Aptana (versió d'Eclipse per a llenguatges a la web), NetBeans, IntelliJ IDEA i Microsoft Visual Studio.
  • Editors en línia. Són aplicacions web que ens permeten escriure i provar el nostre HTML. Per exemple, l'editor de w3schools:
    http://www.w3schools.com/html/tryit.asp?filename=tryhtml_basic
    o bien
    http://htmledit.squarefree.com/
    Per a petits fragments es fa servir https://codepen.io
  • Editors visuals. Permeten treballar de manera que puguem construir pàgines web sense escriure codi, de manera visual. Es basen en el programari WYSIWYG (What You See Is What You Get), el que es veuen a la pantalla és el resultat que s'obtindrà. Referit al fet que treballes veient la feina tal com quedarà finalment. En realitat és impossible ja que l'aspecte final dependrà del dispositiu i el programari que s'utilitzi en veure la pàgina i això variarà enormement d'un usuari a un altre; però com a mínim permeten fer-se una idea molt visual del resultat.
    Adobe Dreamweaver seria el més famós d'aquest tipus i el més complet, ja que també permet treballar còmodament amb el codi. Altres opcions són Adobe Muse, Tumult Hype, Google Web Designer, Kompozer, WYSIWYG Web Builder o Xara Web Designer.

Versions d'HTML

Els estàndards que segueixen en funcionament actualment són els següents:

HTML 4.01

Es tracta de la versió estàndard de l'HTML tradicional feta l'any 1999 i que segueix tenint certa vigència actualment, ja que és l'últim HTML l'estàndard del qual va definir la W3C abans de XHTML.

Per avisar que el nostre document segueix les normes d'HTML 4.01, es col·loca a les pàgines web, una etiqueta DOCTYPE (ha de ser la primera línia del document) que serveix per avisar precisament de quin tipus d'HTML segueix el document i quin document oficial li valida. Per a HTML 4.01 transicional (hi ha una versió d'HTML4.01 estricta i una altra per a les, ja en desús, pàgines de marcs) és:

<!DOCTYPE HTML PUBLIC
  "-//W3C//DTD HTML 4.01 Transitional//EN"
  "http://www.w3.org/TR/html4/loose.dtd">


XHTML

S'ideà com el substitut d'HTML, ja que es considerava que XML quedaria com a llenguatge de marques definitiu i que calia compatibilitzar HTML amb les normes d'XML. La primera versió, la 1.0 segueix sent la més utilitzada. Hi ha també una versió estricta, una transicional i una versió per a pàgines amb marcs (que gairebé no es fa servir). La més utilitzada és la transicional (igual que a HTML 4) perquè permet l'ús de nombrosos atributs i elements que es consideren obsolets, però que molts dissenyadors utilitzen.

Les pàgines XHTML obliguen a que l'escriptura d'HTML segueixi les regles del llenguatge XML ben format, per la qual cosa són més estrictes en la manera d'escriure.

Per indicar que utilitzem XHTML 1.0 transicional cal posar aquesta etiqueta DOCTYPE al principi del document:

<!DOCTYPE html PUBLIC
"-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">


A més, l'etiqueta arrel html de tot document HTML s'ha d'usar d'aquesta manera (especificant l'espai de noms i l'idioma de la pàgina):

<html xmlns="http://www.w3.org/1999/xhtml"xml:lang="ca">


En el cas de XHTML 1.0 estricte, el DOCTYPE seria:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">


L'etiqueta arrel html seria igual que a la versió transicional

HTML 5

Es considera el estàndard actual. A poc a poc s'està convertint en HTML més utilitzat. Es va crear fora del World Wide Web Consortium a l'entitat anomenada WHATWG (Web Hypertext Application Technology Working Group), afavorida per Apple, Google i Mozilla entre altres empreses a les quals s'han anat afegint moltes més.


La W3C va decidir unir-se a la WHATWG el 2008 per buscar un estàndard. Actualment HTML 5 ja és una recomanació oficial de la W3C que ha substituït WHATWG com a organisme encarregat de vetllar per l'estàndard.

HTML5 segueix una filosofia d'"estàndard viu" en què ja hi ha una recomanació d'ús per part seva, però que va canviant contínuament segons es consideren noves millores o l'aparició de noves tecnologies; aquesta forma dinàmica de fer servir les recomanacions ha estat la clau de l'èxit d'HTML 5 des de l'any 2012 (especialment gràcies que els dispositius Apple i Android sempre han manejat molt bé HTML 5).

Des del 2014 és la recomanació oficial de la W3C.

HTML 5 segueix creixent i per això (seguint la filosofia d'estàndard dinàmic de la WHATWG) les pàgines HTML 5 indiquen que usen aquesta versió d'HTML mitjançant un DOCTYPE molt més senzill: 

<!DOCTYPE html>
<html lang="es">


L'etiqueta arrel HTML també és més senzilla que les utilitzades en XHTML (indicant simplement l'idioma).

La versión actual és l'estàndard HTML 5.3

Validar el cod HTML

Els navegadors no són estrictes amb les normes HTML; és a dir, encara que tinguem algunes fallades, els ignoraran per mostrar la millor versió del codi que fem. Als navegadors els interessa fer funcionar el codi, sigui vàlid o no.

Tot i això, com a creadors de pàgines web, hem de complir les normes del llenguatge aprenent així realment el llenguatge ia més impedint que el navegador s'equivoqui en interpretar les nostres fallades. A més, fa que altres creadors de pàgines comprenguin el codi perfectament, facilitant el treball col·laboratiu.

El validador oficial de la W3C (http://validator.w3.org/) permet validar totes les versions HTML, inclosa la versió 5.

Els errors greus es mostren en vermell i hi pot haver avisos (warnings) que són errors lleus, però que cal tenir en compte.

D'altra banda, la major part dels entorns de treball disposa de validador incorporat (al mateix entorn es marquen els errors alhora que escrivim el codi) i alguns altres es pot instal·lar un plugin.

També podem instal·lar validadors en línia d'ordres com l'htmltidy, disponible a http://www.html-tidy.org/, que permeten validar des de la línia d'ordres, encara que aquesta opció cada cop es fa servir menys.

Compatibilitat entre navegadors

La tecnologia dominant actualment per crear pàgines web és la que sorgeix dHTML5 i que inclou lús del llenguatge CSS (versió 3) i de JavaScript.

Tot i que avui dia gairebé tots els navegadors suporten HTML5, molts (sobretot si no estan actualitzats) no suporten totes les característiques. De fet, com que HTML5 és viu i segueix introduint noves possibilitats, no hi ha cap navegador que inclogui totes les opcions.

Tradicionalment, Internet Explorer no és un navegador respectuós amb HTML 5, de fet només a partir de la versió 9 suporta els elements fonamentals d'HTML5 i només a partir de la 10 ho fa de manera realment comparable a la de la resta de navegadors actuals. De fet, o es recomana als usuaris actuals fer servir aquest navegador, excepte ara mostrar aplicacions web realitzades fa anys i que només són compatibles amb aquest, antany dominant, navegador.

El nou navegador de Microsoft, Edge sí que és més compatible. Els navegadors en, general Chrome i Firefox són els que adopten més ràpid les noves normes i aporten més compatibilitat amb HTML5, però depèn de propietats. Altres navegadors com Safari i Opera reconeixen també la immensa majoria d'HTML5.

La pàgina caniuse.com és la referència més completa i utilitzada per saber quines característiques dels llenguatges utilitzats a les pàgines web són suportades a cada navegador.


D'altra banda, el lloc http://html5test.com/ realitza un test al navegador amb què entrem, per determinar el grau en què suporta HTML 5 (li dóna una puntuació) i ens indica quines característiques compleix i quines no ( cal tenir en compte que la tecnologia HTML5 no només es refereix al propi HTML sinó al CSS i JavaScript relacionat, així com a altres tecnologies dús amb HTML).

Una altra opció és http://fmbip.com/ que mostra les capacitats relacionades amb HTML5 que té el nostre navegador.

Una altra possibilitat interessant és utilitzar la llibreria Modernizr (disponible a http://modernizr.com/) que permet als desenvolupadors a JavaScript detectar les característiques que té el navegador de l'usuari i actuar en conseqüència des del codi mateix.

Publicació de llocs web

S'anomena lloc web el conjunt de pàgines web i recursos que contenen tota la informació associada a una determinada adreça d'inici a Internet.


Quan una persona desitja crear un nou lloc web, inicialment us crea al vostre ordinador de treball i per això ha de crear una carpeta i en ella emmagatzemar totes les pàgines i recursos necessaris (imatges, sons, vídeos, arxius auxiliars,…). Aquesta carpeta s'haurà d'enviar al servidor web que hàgim contractat o del que disposem per publicar la nostra pàgina a Internet.

Per això normalment s'utilitza el protocol de transmissió de fitxers conegut com a FTP, encara que és possible que aquesta transmissió es faça amb altres protocols com WebDAV o RDS. Amb copiar la carpeta al lloc adequat del nostre servidor, la pàgina estarà publicada. Normalment per això se'ns demana un usuari i una contrasenya que verifica que realment som els propietaris de l'espai.

Una altra opció, més habitual avui dia per als desenvolupadors, és utilitzar sistemes de control de versions al núvol (el més conegut és GitHub) i descarregar l'última versió del codi directament al servidor web. És a dir, des del nostre ordinador pugem la darrera versió del nostre codi al servidor al núvol i després actualitzem el codi al servidor web.


Edició bàsica d'elements

Etiquetes i elements

Un document HTML conté text. Ara bé, hi ha text que serveix per indicar elements i propietats de la pàgina. És a dir, el text conté el que s'anomena etiquetes.

Les etiquetes serveixen per delimitar elements de la pàgina. Els elements són cadascun dels elements de la mateixa; per exemple, un paràgraf és un element de la pàgina, una taula també. Fins i tot hi ha elements que contenen altres elements (les taules consten de files i les files de cel·les, per exemple).

Les etiquetes són textos tancats entre els signes de més gran i més petit (< >). Quan un navegador es troba un text tancat així, entendrà que dins dels símbols més petit i més gran el que s'indica és l'inici d'un element. L'inici de l'ítem es marca amb el nom de l'ítem entre els símbols < i >. Exemple:

<strong>


L'etiqueta strong serveix per indicar text important (se sol mostrar en negreta)

La majoria d'etiquetes afecten un text determinat, el qual estarà tancat per les etiquetes. Per tant, gairebé sempre hi ha una etiqueta d'obertura i una altra de tancament, que s'interpretarà com l'inici i la fi d'un determinat element de la pàgina.

El tancament d‟una etiqueta es marca posant el símbol /, seguit del nom de l‟element estem tancant. Exemple:

Text normal <strong>text negreta</strong>

Literalment el que indiquem a l'exemple anterior és que el text conté un element de tipus strong, que conté el text: text negreta. Un navegador reaccionarà mostrant aquest text amb un tipus de lletra més fort. És a dir:

Text normal text negreta

Ja s'ha comentat que un element pot contenir dins més elements, per la qual cosa entre l'obertura i el tancament d'una etiqueta hi pot haver més etiquetes. Una norma important és que sempre hem de tancar primer les etiquetes obertes més tard.

És a dir, aquest codi no és correcte.

Text normal <strong><em>text negreta i cursiva</strong></em>

S'està tancant l'etiqueta em després de strong i això no és correcte perquè curs és la darrera que es va obrir. El correcte és:

Text normal <strong><em>text negreta i cursiva</em></strong>


És a dir, primer es tanca l'element em que va ser l'últim a obrir-se.


La veritat és que els navegadors no solen donar problemes quan cometem errors al codi. Fins i tot un error com el que mostra abans, no seria considerat. El que fan és arreglar els nostres errors, sempre que siga possible. La raó és que els interessa que les pàgines es vegen correctament i no actuar com a validadors del codi.

Atributs

Algunes etiquetes tenen atributs. Els atributs són propietats de cada element a què podem assignar un valor, de manera que aquest valor varia el comportament de l'element. La manera d'utilitzar atributs és:

<element nomDeAtribut1="valor1" nomDeAtribut2="valor1" … >


Als atributs se'ls assignen valors que han d'anar entre cometes. El símbol d'igualtat (=) és obligatori.
Exemple:

<p lang="es">


Indica que el elemento está marcado en idioma español. La sintaxis de atributos es:

<etiqueta atributo1="valor1" atributo2="valor2 valor3" ...>
....
</etiqueta>


Noteu que un atribut pot tindre més d'un valor, encara que no és el més habitual. En aquest cas els valors van separats per espais en blanc

Estructura bàsica d'un document HTML

Per crear una pàgina web usant HTML 5, l'estructura bàsica és la següent: 

<!DOCTYPE html>
<html lang="ca">
    <head>
        <meta charset="UTF-8" >
            <title></title>
    </head>
    <body>
        
    </body>
</html>

On el significat dels elements és el següent:

  • línia DOCTYPE. L'etiqueta DOCTYPE indica el tipus d'HTML que estem fent servir (concretament HTML 5). Es tracta d'una línia que a l'actualitat segueix ignorant gairebé tots els navegadors, però que és molt recomanable (i molt respectuós amb la norma) escriure. S'hi indica la versió HTML que segueix el document.
  • etiqueta html. Tot document HTML està tancat dins de l'obertura i tancament d'aquesta etiqueta. Marca'n el principi i la fi (marca el element arrel d'un document HTML). Cal afegir l'atribut lang per indicar l'idioma en què està escrit el document.
  • capçalera, etiqueta head. Tanca les etiquetes de capçalera, les quals proporcionen informació que el navegador ha de tenir en compte per al contingut del document aparegui de forma correcta i també conté elements que proporcionen informació als cercadors (com Google). Els elements habituals que conté són:
    • etiquetes meta . Que serveixen per indicar aspectes de funcionament de la pàgina web com, capçaleres del protocol http, indicacions sobre cada quant caduca la pàgina, paraules clau per buscar la pàgina als navegadors, codificació de caràcters de la pàgina,… En aquest cas només s'usa l'etiqueta meta obligatòria que indica que estem codificant el text usant el format utf-8 de Unicode.
    • títol, etiqueta title. El títol de la pàgina, que els navegadors solen col·locar a la barra de títol de la finestra on es mostra la pàgina.
    • declaració de fitxers externs i scripts. A l'exemple no hi ha declaracions d'aquest tipus.
  • cos. Tancat a l'etiqueta body , tanca el contingut en si de la pàgina web. El que el navegador mostra per pantalla és el que es col·loca en aquest apartat.

Espais en blanc

Funcionament dels espais en blanc

El text dins de les pàgines web no respecta els espais en blanc ni tabuladors que col·loquem al codi, a l'hora de mostrar el contingut per pantalla. Només es considera el primer espai en blanc, la resta s'eliminen.

Per exemple:

<!DOCTYPE html>
<html lang="ca">
    <head>
        <meta charset="UTF-8">
            <title></title>
    </head>
    <body>
        Aquest text
                                en té molts
                         espais
    </body>
</html>


El resultat mostraria a la pàgina aquest text:

Aquest text té molts espais

La raó és permetre una major llegibilitat del codi, permetent-hi que col·loquem el codi amb sagnies i espais que millorin la seva lectura, perquè el manteniment del mateix sigui més còmode.

Etiqueta pre

Si volem que els espais que col·loquem al codi sí que siguin tinguts en compte, podem utilitzar l'etiqueta pre .

Realment, amb caràcter general, no s'aconsella l'ús de pre perquè ajuda a adquirir mals hàbits i dificulta l'aprenentatge de les formes avançades de maquetar pàgines web.

Sí que hi ha contextos en què és obligat el seu ús, per exemple per mostrar textos referits a exemples de codi de llenguatges de programació.
Exemple: 

<!DOCTYPE html>
<html lang="ca">
    <head>
        <meta charset="UTF-8">
            <title>Hola món</title>
    </head>
    <body>
                <p>Programa <strong>Hola món</strong>
                escrit amb Java

    </body>
</html>
<pre>
public class HolaMon{
    public string void main(String[] args){
        System.out.println("Hola Món");
    }
}
</pre>


Com que la pàgina anterior mostra un exemple de codi en Java, el més apropiat és utilitzar pre per mantenir les tabulacions tal com es van fer en escriure el codi. El resultat seria:

Programa Hola món escrit amb Java

public class HolaMon{
    public string void main(String[] args){
        System.out.println("Hola Món");
    }
}

Ús de &nbsp;

Una altra manera d'obligar a tenir en compte espais en blanc és el codi  . Aquest codi és una entitat (les entitats s'expliquen més endavant en aquest capítol). Si, per exemple, escrivim quatre vegades seguides &nbsp; estarem deixant quatre espais en blanc que el navegador sí que tindrà en compte. Exemple:

text &nbsp; &nbsp; &nbsp; &nbsp; un altre text

El segon text apareixerà quatre espais en blanc darrere del primer.

text         un altre text

Codificació de caracters en HTML

Ús de l'etiqueta meta charset

Els documents de text es poden codificar de diferents maneres. Exemples d'aquestes formes són la codificació ASCII clàssica, o la taula ISO 8859-1 que és una taula ASCII estesa pensada per a les llengües d'Europa Occidental.

Cal tenir en compte que les pàgines web es poden veure, un cop publicades a Internet, arreu del món i el mateix codi es pot interpretar de manera diferent si no avisem de la manera com hem codificat el document.

Fins fa uns anys, per escriure en espanyol s'aconsellava fer servir la codificació ISO-8859-1 (encara hi ha milions de pàgines que usen aquesta codificació), però ara el recomanable és fer servir Unicode, concretament en la seva forma UTF-8. Per indicar que la nostra pàgina utilitza Unicode UTF-8 només cal indicar a la capçalera, l'etiqueta:

Indicació en el programari de escriptura de la pàgina

No val només fer aquesta indicació. Hem d'estar segurs que realment estem codificant a Unicode. Per exemple si fem servir el Bloc de notes de Windows per escriure la pàgina web i guardem sense cura, no estarem codificant el text a Unicode. Cal estar segurs que en desar estem realment guardant el text usant Unicode UTF-8:

Si no assegurem que estem codificant així, en obrir la pàgina no es mostraran bé els caràcters fora de l'ASCII, quan la persona que veu la nostra pàgina utilitza un altre sistema diferent del nostre (per exemple si obre la pàgina en un sistema Linux i nosaltres havíem fet servir la codificació nativa de Windows).

Avui dia tots els editors professionals guarden de forma nativa a UTF8, però no està de més assegurar-nos.

Ús d'entitats

Introducció a les entitats

Per evitar els problemes de la codificació, una altra opció (menys recomanable avui dia) és només utilitzar caràcters dins de l'ASCII original. Els caràcters fora de l'ASCII original s'escriurien usant les entitats.

A més, hi ha símbols que no poden ser escrits com a text normal ja que es malinterpretarien pels navegadors com els símbols > o < per exemple.

El problema és que l'ASCII no permet símbols d'altres idiomes com l'eñi i per això es va idear poder escriure codis nacionals usant una notació especial. Són les entitats.

Les entitats són codis especials que comencen amb el símbol & després del qual s'indica el nom de l'entitat (un nom mnemotècnic) i final es delimita amb un punt i coma (;).
Per exemple, &ntilde; és l'entitat que representa l'eñe. És el mateix escriure ñ que escriure &ntilde;

També és possible codificar utilitzant la notació: &#n;

On n és el número del codi dins de la taula Unicode (cal recordar que els primers 256 caràcters són els corresponents a la taula ISO-8859_1).

Evidentment és molt pesat escriure tots els símbols nacionals usant entitats (encara que hi ha programari que és capaç de fer la traducció automàticament a mesura que escrivim el text), per això i perquè ara ja està molt implantat Unicode per codificar el text, ja no es fan servir les entitats amb aquest propòsit.

Codis d'idioma

Com hem indicat anteriorment, l'atribut lang permet indicar dues lletres que signifiquen l'idioma en què està escrit el text de l'element. El més normal és indicar-lo a l'element html (per exemple <html lang="ca">), però hi ha vegades que una pàgina conté elements el contingut dels quals està escrit en un idioma diferent de l'indicat com a general a l'apartat <html>. Com l'atribut lang, és comú a tots els elements, es pot indicar quan sigui necessari. Exemple:

<p>
Com dirien als Estats Units:
<span lan="en">Welcome!</span>
</p>


Per què prendre tantes molèsties? Perquè, per exemple, els cercadors d'Internet distingeixin el text en l'idioma apropiat i adaptin millor les nostres cerques. Altres eines (com a correctors ortogràfics, traductors en línia, etc.) funcionen molt millor marcant adequadament l'idioma.

Podem ser més específics en l'idioma i indicar no només l'idioma sinó el dialecte concret. S'acompanya d'un guió al codi de l'idioma seguit de dues lletres majúscules. La norma ISO que conté els codis disponibles és la norma 639-2.

Per exemple, si volem marcar un element amb l'idioma espanyol, però indicant que és espanyol de Mèxic, faríem el següent:

<p>
    Com dirien a Mèxic:
    <span lan="es-MX">¡Hijole!</span>
</p>


Contingut i contenidors

En HTML els elements que mostren alguna cosa a la pàgina, es col·loquen dins de la secció body.

A més, cal tenir en compte que hi ha elements pensats per contenir grans quantitats d'informació (com div ) i elements que contenen poc text (com strong). És correcte que aparegui un element strong dins un element p, però no al revés:

<strong><p>Hola</p></strong> -->¡Incorrecte!!
<p><strong>Hola</strong></p> -->Correcte


La majoria d'elements HTML són contenidors d'altres textos i elements. Cal conèixer quins tipus d'elements tenim. Els tipus estan relacionats amb la propietat CSS display, que s'encarrega de dir com s'ha de mostrar un element. Per defecte, els elements funcionen d'aquesta manera:

  • Elements de tipus inline. Són elements que es mostren seguits a la mateixa línia. Aquests elements són els que marquen text simple. Totes les etiquetes de marcatge de text tenen aquest display: strong, em, span, mark, abbr, etc.

    Exemple, el codi:

    <strong>Hola</strong> <em>Hola</em>

    Es mostra així:

    Hola Hola
  • Elements de tipus block. Són contenidors més grans. Els elements que marquen paràgrafs sencers (com p, h1, h2 o blockquote) i els contenidors de seccions (com div, section, article o figure) tenen aquest display. La seva característica és que, per defecte, tenen forma rectangular i abasten tota l'amplada del contenidor. Si indiquem per a ells un color de fons, aquest es mostra a forma de rectangle (en els inline, el color de fons abasta exactament el text, com si ho haguéssim pintat amb un retolador). Exemple: 
    <h1>Hola</h1> <p>Hola</p>

    Aquest codi es mostra així:

    Hola
    Hola
  • Elements de tipus inline-block. Són rectangulars com els block, però no abasten tot el seu contenidor. Es adapten al contingut. Exemples són els elements button o imatge: 
    <strong>Hola</strong> <button>Hola</button>

    El resultat mostra un botó (element rectangular) seguit del text de tipus strong. Si el botó tingués un display de tipus block, saltaria a la línia següent.

    Hola

En tot cas, el display d'un element es pot modificar des del llenguatge CSS, però no convé fer combinacions antinaturals ja que es perd el sentit semàntic del codi.

Atributs comuns

Tots els elements tenen una sèrie d'atributs comuns a tots. Això vol dir que els podem utilitzar independentment de quin sigui l'element.

L'atribut comú més conegut és id , que permet identificar un element amb un identificador únic, però n'hi ha molts més. Els principals són:

atribut

significat

lang

Indica el llenguatge en què està escrit el text de l'element

style

Permet indicar codi CSS directament. Aquest codi només afectarà aquest element.

class

Permet indicar una classe CSS per a l'element

id

Permet identificar l'element. Un identificador és un nom únic que se li dóna al element i que no es pot repetir en un altre element dins de la mateixa pàgina web.

L'identificador només pot contenir lletres (de l'alfabet anglès) i números (no es poden utilitzar espais en blanc). A més, ha de començar per una lletra.

L'atribut id és un dels fonamentals.

Permet posar un títol a l'element. És una mena de descripció curta que és molt útil en aquells elements que requereixen una explicació semàntica com un abreviatura, una sigla, una imatge, etc. Normalment, els navegadors reaccionen a aquest atribut mostrant un cartell amb el text indicat a title, quan el cursor del ratolí s'aproxima a un element que utilitzi aquest atribut

Introducció

Avui dia en una pàgina web, tot el text és dins d'algun element (delimitats mitjançant etiquetes), en realitat hi ha tres contextos principals a tenir en compte sobre el text

  • Elements capaços de contenir diversos paràgrafs. La W3C anomena aquests elements sectioning content. Cada element d'aquesta part és el que avui dia es coneix com a capa o secció. Si no indiquem capa alguna, llavors tots els paràgrafs estan continguts a l'etiqueta body, cosa que significa que el contenedor dels paràgrafs serà l'àrea que el navegador deixa per al text i que pot ser gairebé tota la pantalla.
  • Elements que contenen paràgrafs individuals. Tot i que a HTML 5 es permet escriure text que no estiga dins un paràgraf, no és una bona pràctica, ja que no podríem indicar més endavant el format del paràgraf (alineacions, marges,…).
    Lògicament dins un paràgraf, no podem indicar un altre paràgraf; és a dir, no podem introduir una etiqueta p, dins una etiqueta h1.
    Quan un text està dins un element de paràgraf (com p ), farà que el text següent quede en un altre paràgraf, per tant hi haurà un salt de paràgraf entre els dos textos.
  • Elements que contenen text simple o conjunts de caràcters. La W3C crida aquests elements phrasing content. Són elements pensats per ser interiors a un paràgraf. Un mateix paràgraf pot contenir molts elements d'aquest tipus. El mateix text pot estar contingut a diversos elements d'aquest tipus alhora.
    Per exemple, podem indicar que un text és strong i em alhora.

Elements de marcat de text simple

Text important

A HTML 5 el text important es marca amb l'element strong. En general, tots els navegadors marquen el text strong en negreta (lletra més gruixuda), però que es mostri en negreta és un acord visual que es pot modificar amb el llenguatge CSS. És a dir, no és el mateix text en negreta que el text important.

Sens dubte, strong és un dels elements HTML fonamentals i imprescindible als documents HTML.

Text emfatitzat

L'element em marca text com a emfatitzat. El resultat visual amb què els donen format per defecte els navegadors és text en cursiva.

La idea semàntica és marcar amb em el text amb èmfasi de segon nivell (deixant a strong el primer nivell) que normalment es dedica a paraules no acceptades encara, títols de treballs, sobrenoms, text subjectiu, etc.

Text ressaltat

Aquest és el tercer element d'HTML5 per remarcar text. Es fa mitjançant l'element mark. En aquest cas, el seu efecte visual sol ser aplicar al text marcat així un fons de color viu que simula l'ús d'un subratllador.
Exemple: 

<p>Jo soc text normal, <mark>i jo estic remarcat
</mark></p>


Obtindria el resultat:

Jo sóc text normal, i jo estic remarcat

Aquest element no és tan antic com els anteriors perquè va aparèixer amb HTML5 (els navegadors antics no ho reconeixen). La idea semàntica és marcar text que volem realçar de manera molt cridanera indicat text a recordar o sobre el qual aprofundir.

Element b

Es tracta de l'element històric (de les primeres versions HTML) utilitzat per marcar text en negreta. En HTML 4 va quedar com a obsolet. En definitiva, element d'ús molt poc aconsellable que només es mantenen per la gran quantitat de creadors que el segueixen utilitzant.

Teniu el mateix problema que l'anterior. És l'element històric (de les primeres versions HTML) utilitzat per marcar text en cursiva. En HTML 4 va quedar com a obsolet. A HTML 5 es pot utilitzar per marcar text tipus alternatiu si no encaixa amb altres elements de marcatge de caràcters.

La Intel·ligència Artificial

La Intel·ligència Artificial (IA)1 és un camp d'estudi que s'enfoca a desenvolupar sistemes i algorismes que poden realitzar tasques que, quan són realitzades per éssers humans, requereixen intel·ligència. Aquestes tasques inclouen l'aprenentatge, la resolució de problemes, el raonament i la presa de decisions.

Aplicacions de la IA

La IA té una àmplia varietat d'aplicacions a la vida quotidiana. Alguns exemples inclouen:

  • Assistents virtuals com Siri2 i Alexa3.

Element div

Es tracta d'un element clàssic, ja era present a les primeres versions d'HTML, que s'utilitza com a contenidor d'altres elements per poder manipular-los tots alhora. Dins de div es poden col·locar tota mena d'etiquetes (taules, paràgrafs, imatges,…).

Des de fa anys ha esdevingut l'element utilitzat, combinat amb CSS, per crear capes. Les capes són elements visuals que es poden posicionar a voluntat a la pàgina web. D'aquesta manera, mitjançant les capes s'aconsegueixen maquetacions molt artístiques.

Per defecte, els navegadors no donen format al contingut d'un element neg, de manera que visualment no es diferencia de la resta d'elements, llevat del fet que un element div té un display de tipus block . És a dir, el contingut d'aquests elements, per defecte, es mostra de forma separada a la resta.

<div style="background-color:red";>
    <h2>Títol secundari</h2>

Element span

És molt similar a l'anterior, però com que els navegadors no afegeixen res (ni tan sols espais) a aquesta etiqueta quan es mostra per pantalla, en lloc de per definir capes (encara que es podria) s'usa per marcar contingut interior a un paràgraf, amb la finalitat de que a aquest contingut se li puga donar un format especial mitjançant CSS. És a dir, amb div marquem capes (contenidors grans) i amb span marquem blocs xicotets de text.

Exemple:

<p>Dins d'aquest text <span style="color:red">aquesta frase surt de color vermell</span>. Això torna a sortir normal</p>
