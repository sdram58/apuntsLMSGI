Introducció als fulls d'estil

Concepte de fulls d'estil

Els fulls d'estil són elements agregats al llenguatge Html que tindran en compte la presentació del document o de l'aplicació Web.


El mateix contingut html es pot presentar de manera diferent segons el full d'estil utilitzada. En aquesta web (css zen garden) es pot vore clarament com canvia una pàgina d'una altra només canviant l'estil

Ús de les fulles d'estil

  • Obligatòries en HTML5: Separació entre contingut i presentació.
  • Simplificació del codi
  • Manteniment del lloc web més senzill: Els fulls d'estil permeten, modificant un únic fitxer, actualitzar la presentació gràfica del conjunt de pàgines d'un lloc complet
  • Una via cap a l'accessibilitat: Els fulls d'estil ofereixen, per exemple, la possibilitat de visualitzar una pàgina de text amb caràcters més grans i, per tant, més llegibles.

Breu Història

  • 1991 Tim Berners- Lee inventa la web, el llenguatge HTML es concep per al intercanvi dinformació entre els científics de tot el món.
  • 1994. Sorgeix la idea de proporcionar estil a les pàgines web
  • 1995. El W3C (World Wide Web Consortium) va començar a funcionar i publica el primer esborrany sobre els fulls d'estil.
  • 1996. L'esborrany es converteix en una recomanació oficial amb el nom "Fulls d'estil en cascada, nivell 1" o CSS1
  • 1998. Es publica una extensió de la recomanació amb el nom CSS2
  • 2006. Es publiquen algunes correccions i extensions baix el nom CSS2.1.
  • 2009. Es va començar a parlar de les fulles d'estil CSS3.

Validador de les fulles d'estil

Igual que per a HTML5, hi ha un validador per als fulls d'estil.

El validador en línia del W3C per als fulls CSS (http://jigsaw.w3.org/css-validator/) és una eina de gran valor amb una qualitat reconeguda unànimement. Permet lliurar pàgines perfectes, respectant escrupolosament les normes dictades pel W3C.


El validador CSS només comprova els fulls d'estil. D'aquesta manera, només cal introduir el fitxer de fulls d'estil extern o el contingut de l'etiqueta <style> per a les fulls d'estil interns del document.

Declaració d'estils i selectors bàsics

La declaració d'un estil es fa mitjançant el binomi propietat: valor: Exemple:

background-color: red;


Selectors d'etiqueta.

La sintaxi d'un selector d'etiqueta és el nom de l'etiqueta seguit de la declaració d'estil entre claus.
selector { declaració d'estil }

Exemple

div { background-color: red;}


Podem aplicar el mateix estil a diferents selectors d'etiqueta separant-los amb comes. Exemple:

h1, h2, h3 {declaració d'estils}


Selectors de classe

El dissenyador defineix els seus propis selectors.
.nom_de_la_classe { declaració(ns) d'estil}

Exemple

.roig { background-color: red;}

La crida a la classe es realitza de la següent manera:

<etiqueta class="nom_de_la_classe">

Exemple

<div class="roig"> ... </div>

Podem definir una classe únicament per a una etiqueta, per exemple

blockquote.roig { background-color: red;}

Amb aquesta definició de classe, només la podrem aplicar a l'etiqueta seleccionada, en aquest cas blockquote.

El primer caràcter de la classe no pot ser un número, un guió o un caràcter de subratllat.

Selectors d'identificador

L'identificador ha de ser únic per a cada element de la pàgina, per tant, el selector d'identificador s'invocarà una sola vegada al document. La definició d'un selector id és:

#nom_del_identificador {declaració d'estil}

Exemple

#roig { background-color: red;}

La cridada es fa mitjançant:

<etiqueta id="nom_d'identificador">

Exemple:

<div id="roig"> ... </div>


Comentaris

En un full d'estil CSS, un comentari comença amb els caràcters d'obertura barra obliqua i asterisc (és a dir /*) i acaba amb els caràcters asterisc i barra obliqua (és a dir */)

Exemple

/* Això és un comentari */

Notacions de colors

Els fulls CSS proporcionen múltiples notacions per declarar un color.

  • La notació hexadecimal clàssica, és a dir #ffcc00:. Defineix el color, o més aviat els tres components del color. roig (r de red), verd (g de green) i blau (b de blue), mitjançant una notació hexadecimal de tipus #RRGGBB.
    • Notació amb alpha: #RRGGBBAA s'hi afegeixen un parell més al final que fa referència al canal alpha
    • Notació curta: podem escriure-ho usant un sol digit per color/alpha. El resultat serà que per a cada color es duplica la lletra. Evidentment és menys precís. #FC0 será igual a #FFCC00
  • La notació decimal, per exemple, color: rgb(0, 0, 255). El codi RGB del color no es codifica segons el valor hexadecimal sinó per un número comprès entre 0 i 255.
    • rgba(R, G, B, A): s'afegiex un quart paràmetre que és el alpha. El grau d'opacitat. Aquest pot estar en percentatge o decimal de 0 a 1, sent 0 no es veu, completament traparent i 1 completament opac.
  • La notació en percentatge:, per exemple, color: rgb(25%, 50%, 0%). El valor 0% significa l'absència del component esmentat i 100% el valor màxim.
  • Paraules clau: per exemple, color: red.
  • La notació HSL (Hue Saturation Luminance), al valencià Tonalitat: Saturació, Lluminositat. La notació HSL consisteix en tres valors.
    • El primer s'expressa en graus, de 0° a 359° (el símbol de grau º no apareix a la notació). Es correspon amb un color de la roda cromàtica: roig (0°), groc (60°) , verd (120°), cian (180°), blau (240°) i magenta (300°)
    • El segon i el tercer valor s'expressen en percentatge i descriuen respectivament la saturació i la lluminositat. Per exemple, color: hsl(0, 100%, 50%) per al roig.

CSS integrades en un element HTML5

A HTML és possible afegir directament al codi una declaració d'estil en una etiqueta determinada.

<h1 style="color: red;">Título de nivel 1</h1>

És convenient evitar aquesta manera de procedir en virtut del principi de separació entre contingut i presentació. Això no obstant, pot resultar útil per donar prioritat a una propietat d'estil respecte a la que s'haja especificat a la declaració interna o externa.

<!DOCTYPE html>
<html lang="ca">

    </head>
    <body>

        <h1>Títol 2</h1>
    </body>
</html>

CSS internes al document HTML5.

L'ús principal dels fulls destil CSS consisteix a determinar un estil que s'aplique a la totalitat d'una pàgina HTML5.

El codi dels fulls d'estil s'agrupa a la capçalera del document, entre les etiquetes <head> i </head>.

La sintaxi es presenta de la següent manera:

<head>
    <style type="text/css"> 

        h1 { color: red;}

    </style>

</head>

CSS externes al document HTML5.

Les declaracions d'estil es poden agrupar en un fitxer extern (diferent) al fitxer Html.

Aquesta manera de fer respecta millor el principi de separació entre contingut i presentació.

Aquesta tècnica empra dos arxius:

  • El full d'estil extern: Un fitxer que conté la declaració d'estil. Aquest fitxer té l'extensió .css.
  • El document Html5: Entre les etiquetes <head> i </head>, s'afegeix un enllaç al full d'estil en qüestió. 
    <link rel="stylesheet" type="text/css" href="exemple.css">

    on:
    • link indica al navegador que el que segueix és un enllaç.
    • rel="stylesheet" necessita que aquest enllaç siga relatiu a un full d'estil.
    • href="exemple.css" és l'escriptura clàssica d'un enllaç a Html.
    • L'enllaç pot ser absolut (començant per http://...) o relatiu.
    • Res no impedeix incloure diverses etiquetes <link> cap a fulls d'estil externs diferents.
    • Res no impedeix utilitzar, a més, un full d'estil intern.

Exemple:

<head>
    <title>Html5</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" type="text/css" href="exemple.css">
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>

CSS importada (@import).

Una altra forma d'usar els fulls d'estil externs consisteix a fer servir l'ordre @import.

<style type="text/css"> 
    @import url(exemple.css);
</style>


@import es podrà utilitzar no només per invocar un full d'estil extern en un document Html sinó també per importar un altre full d'estil al full d'estil extern.

<!DOCTYPE html>
<html lang="ca">
<head>
    <title>Html5</title>
    <meta charset="UTF-8">
    <style type="text/css">
        @import url(exemple.css);
    </style>
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>


Les etiquetes <style> ... </style> són necessàries.

Noció de cascada

Als apartats anteriors hem constatat com és possible tindre diverses definicions d'estil, bé en línia, interna(es) o externa(es). n cas que hi haja una competència entre diversos elements d'estil, intervé la noció de cascada (la paraula Cascading a Cascading Style Sheets) o ordre de prioritat.

El navegador pren primer en consideració les especificacions dels fulls d'estil externs (amb l'extensió css), a continuació els fulls d'estil interns (aquells situats a l'interior de les etiquetes <head>) i, tot seguit, els fulls d'estil en línia (aquelles lligades a un element Html5).

L'ordre creixent de prioritat (de menor a major) és:

  1. Propietats per defecte del navegador.
  2. Fulles d'estil externes.
  3. Fulles d'estil internes.
  4. Fulls destil en línia.

La regla de prioritat, per visualitzar el document al navegador, consisteix en utilitzar el full d'estil més proper a al element.

Exemple:

L'arxiu d'estil extern estil1.css es:

h1 { text-decoration: underline;}

<!DOCTYPE html>
<html lang="ca">
<head>
    <title>Html5</title>
    <meta charset="UTF-8">
        <link rel="stylesheet" type="text/css" href="estil1.css">
    <style type="text/css">
        h1 { text-decoration: overline;}
    </style>
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>
</html>

Donant com a resultat:

Títol de nivell 1

El fitxer d'estil extern (estil2.css) és:

El document Html5:

Noció d'herència

Si tenim diverses etiquetes anidades amb diferents estils


Els estils s'hereten:

  • L'estil 2 es caracteritza pel propi estil més el de l'estil 1, que ha heretat.
  • L'estil 3 es caracteritza pel propi estil més el dels estils 1 i 2, que ha heretat.
  • Seguint la mateixa lògica podem utilitzar els termes "pare" i "fill".
  • L'estil 1 seria el pare de l'estil 2. De la mateixa manera, l'estil 2 i l'estil 1 serien pares de l'estil 3. O bé l'estil 2 és fill de l'estil 1 i l 'estil 3 és fill dels estils 2 i 1 .

Selectors

Selector universal

Permet aplicar un estil a totes les etiquetes del document. Això ho aconsegueix l'asterisc. Exemple:

* {
    color:black;
}

No se sol utilitzar així ja que és massa indiscriminada. Però sí que s'utilitza per a elements d'aquest tipus:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        table *{
            color:red;
        }
    </style>
</head>
<body>
    <p>Ix normal</p>
    <table>
        <tr>
            <th>Ix roig</th>
        </tr>
        <tr>
            <td>Ix roig</td>
        </tr>
    </table>
</body>
</html>


Permet pintar en roig el text que estiga dins una taula; no importarà que etiqueta siga la que està dins de la taula. Un altre exemple:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p *{ color:red;}
    </style>
</head>
<body>
    <p>Ix normal <span>Ix roig</span></p>
</body>
</html>


Ara ix de color roig el text dins de l'etiqueta span.

Selectors per atributs

Permet aplicar estils a un element quan aquest té un valor determinat sobre un atribut. Per fer-ho, s'indica l'atribut entre claudàtors [], seguit del signe d'igualtat i el valor entre cometes. Exemple:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p[lang="en"]{
            font-style: italic;
        }
    </style>
 </head>
 <body>
     <p lang="es">text que ix normalment</p>
     <p lang="en">text que ix en cursiva</p>
 </body>
 </html>

Es pot barrejar aquest tipus de definicions amb classes o definicions per identificador:

p.classe1[lang="en"]{
    font-style: italic;
}


En aquest cas, l'estil definit s'aplica a paràgrafs de classe 1 que estiguen marcats amb el valor a l'atribut lang (és a dir, que estiguen en anglès).

Fins i tot es pot utilitzar més d'un atribut:

p[lang="en"][spellcheck="true"]{
    font-style: italic;
}

En aquest cas s'aplica l'estil per als elements de tipus p que utilitzen els atributs lang i spellcheck amb els valors indicats.

Fins i tot podem utilitzar diversos valors per al mateix atribut:

p[lang="en"][lang="fr"]{
    font-style: italic;
}


Aplica la cursiva a paràgrafs el llenguatge dels quals siga anglès o francès.

També podem indicar l'estil simplement per als elements que usen l'atribut independentment del valor:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p[lang]{
            font-style: italic;
        }
    </style>
</head>
<body>
    <p>Aquest text es mostra normalment</p>
    <p lang="en">This text is shown with italics (es mostra amb cursiva)</p>
    <p lang="ca">Aquest text també es mostra en cursiva</p>
</body>
</html>

Altres selectors jeràrquics

sintaxis

significado

element1 ~ element2

S'aplica al element2 quan és germà del element1 i aquest el precedeix, encara que no siga immediatament.

element:empty

S'aplica quan l'element és buit

element:nth-child(número)

S'aplica a l'element indicat quan siga el fill amb el número indicat (per exemple, número seria 3, per al tercer fill).

Es poden fer servir expressions més complexes mitjançant l'ús de la variable n per aconseguir fórmules més complexes (la n el navegador la substituirà automàticament per cada número sencer per calcular quines files se seleccionen)

Es permet també utilitza les paraules clau odd (imparell) i even (parell)

Exemples dús:

  • tr:nth-child(3). Selecciona la tercera fila d'una taula
  • tr:nth-child(odd). Selecciona les files senars
  • td:nth-child(even). Als parells
  • tr:nth-child(2n+1). Selecciona les files 1,3,5,... (les senars)
  • tr:nth-child(3n+1). Selecciona les files 1, 4, 7, 10, 13... (de tres a tres)

element:nth-last-child(n)

Funciona igual que l'anterior però compta els elements fills a partir de l'últim (és a dir, l'ordre del darrere cap endavant).

element:first-child

S'aplica a l'element quan és el primer fill

element:last-child

S'aplica a l'element quan és el darrer fill

element:only-child

S'aplica quan l'element és l'únic fill

element:first-of-type

Primer descendiente de su tipo

element:last-of-type

Últim descendent del seu fill

element:nth-of-type(n)

Funciona com nth-child, però fa referència al número de fill d'aquest tipus. No compte fills, sinó fills del tipus indicat.

Exemple:

<head>
    <style>
        article p:nth-of-type(2){
            color:red;
        }
    </style>
</head>
<body>
    <article>
        <h1>Primer fill, no s'acoloreix</h1>
        <p>Segon fill, primer de tipus p. No s'acoloreix</p>
        <p>Tercer fill, segon de tipus p. ES COL·LOREJA</p>
        <p>Quart fill, tercer de tipus p. No s'acoloreix</p>
    </article>
</body>

element:nth-last-of-type(n)

Com l'anterior però conta n des del final.

element:only-of-type

Pseudoclasses

Bàsiques

Les pseudoclases permeten associar estils a un selector quan li passa una determinada circumstància. Inicialment les pseudoclasses s'aplicaven als enllaços (element a), però ara s'apliquen a qualsevol element. Les pseudoclasses clàssiques són:

  • :link. S'aplica als enllaços no visitats
  • :visited. Enllaços visitats
  • :activeu. Enllaços actius (aquells sobre els quals fem clic)
  • :hover. S'aplica quan el ratolí passa per sobre de l'element (seguiu un enllaç o no)

Exemple: 

a{
    text-decoration: none;
}
a:visited{
    color: gray;
}
a:hover{
    text-decoration: underline;
}

A l'exemple els enllaços als quals s'aplique aquest full d'estil sortiran sense subratllar, si estan visitats sortiran de color gris i si s'hi acosta el ratolí serà quan es mostren subratllats.

Aquestes pseudoclases permeten un cert dinamisme a la pàgina HTML. Actualment, a més, és possible utilitzar-les en altres elements diferents de l'etiqueta a (com p, div, etc.), cosa que dóna enormes possibilitats.

CSS3 als navegadors

Fins a la versió 9, Internet Explorer no ha aplicat els selectors avançats pertanyents a CSS3. La resta de navegadors els va incorporar molt abans.

No és l'únic cas d'aplicació tardana. Internet Explorer és, sens dubte, el navegador que més triga a incorporar millores de CSS3 que els altres navegadors porten implementant des de fa un temps. Però la resta també triguen a aplicar aquestes millores de CSS (lògic ja que CSS3 no deixa de créixer).

El seu successor, Edge, és més respectuós amb el CSS3. Però la qüestió és que encara hi ha molts usuaris que utilitzen Internet Explorer.

També pot passar que alguns navegadors accepten algunes coses i d'altres no. Per exemple que accepten la pseudoclasse avançada ::selection i tanmateix no reconeguen :not. Això complica enormement la feina del dissenyador de pàgines web.

Les últimes versions de tots els navegadors ja incorporen la majoria dels selectors.

Tot i que de vegades cal comprovar que no es requereix prefix. És el cas del selector ::selection, només els últims navegadors el reconeixen i en el cas del Firefox cal acompanyar-lo del prefix –moz-

Ejemplo:

p::-moz-selection{
    background-color:red;
}

En definitiva, cal fer servir amb cautela aquest tipus de selectors.

A causa de la manca de compatibilitat tan gran d'Internet Explorer amb CSS3, és habitual utilitzar codi concret per a aquest navegador (i que només volem que execute aquest navegador) per a això s'utilitza aquest truc:

<!--[if lt IE 9]>
    ..codi ignorat per tots els navegadors excepte per Internet Explorer anteriors a la versió 9
<![endif]-->

    Propietats de les fonts


    Indubtablement la tipografia (anomenada també format de font o, simplement, font) és un dels aspectes més importants de l'estètica d'un document escrit. Manejar adequadament l'estil de la lletra pot fer que la nostra pàgina siga molt més atractiva. De fet, hi ha dissenyadors que consideren que aquesta és la primera decisió a prendre per la tremenda influència que té sobre la resta d'elements. L'ideal és dedicar molt de temps a aquesta decisió.

    CSS disposa de nombroses propietats per modificar els tipus de lletra. Moltes de les seves propietats es poden observar a la imatge anterior s'observen les línies bàsiques que permeten encaixar les lletres. Els elements que cal tindre en compte són:

    • Línia base. És la línia principal on es col·loquen les lletres. És la línia que permet col·locar adequadament a l'horitzontal el text.
    • Alçada de la lletra x. Mesura la mida de la lletra equis minúscula, que serveix de referència per a totes les lletres minúscules (encara que algunes com la g i la del primer exemple la puguin excedir).
    • Ascendent. Línia superior que indica el màxim que pot ocupar les lletres (a la imatge és el límit superior de la lletra f ).
    • Descendent. Línia inferior, posició màxima inferior que pot ocupar el text (la lletra g a l'exemple és la que marca aquesta posició)
    • Alçada majúscules. Línia superior màxima de les lletres majúscules (se sol prendre la de la lletra M majúscula).
    • Interlineat. Distància entre dues línies, es pren utilitzant les línies base.
    • Interletrat. Distància horitzontal entre lletra i lletra. Ampliar significaria distàncies més cada lletra d'una paraula.
    • Kerning. És la distància entre lletra i lletra però aplicant distàncies diferents en funció de les lletres. Així a la imatge anterior s'observa que la distància entre la M i la lletra o , no és la mateixa que entre la o i la x (la x es fica una mica més dins de la o ), ja que així el text és més estètic.
    • Cos. La major part de les tipografies actuals tenen mides (en horitzontal) diferents de les lletres. És a dir, no ocupa el mateix la lletra M que la lletra N (la N és més curta). Els anomenats tipus de lletra monoespaiats (com la lletra Courier ), fan que cada caràcter ocupi el mateix (a l'estil de les màquines d'escriure antigues) la M ocupa el mateix que la N .
    • Lletres vectorials i lletres en mapa de bits. Avui dia totes les tipografies són de tipus vectorial, això vol dir que no es guarda una imatge de cada lletra, sinó que de cada lletra es guarda la fórmula per dibuixar-la. Això permet que en ampliar les lletres mai perdem resolució (al contrari del que passava als ordinadors antics).

      Les primeres lletres vectorials van ser les pertanyents al llenguatge PostScript d'Adobe inclosos als ordinadors Apple MacIntosch. Microsoft va crear el tipus de lletra TrueType per a Windows i les dues companyies actualment desenvolupen els tipus OpenType que s'espera es converteixi en un estàndard obert.

      Actualment es dóna per fet que totes les fonts són vectorials.
    • Família. Hi ha dues grans famílies de lletres les lletres de tipus Serif i les Sans-Serif. Les primeres es distingeixen per col·locar potes en algunes lletres. És el cas de la lletra Times, que escriuria així: MÈTODE (observeu-vos les potes de la M i la T ), mentre que en forma sans serif, per exemple com la lletra Arial, s'escriuria així: MÈTODE.
      • A més d'aquestes dues famoses famílies, CSS permet distingir tres famílies més. En total són:
    • Serif. Ja explicada, per exemple: Times , Garamond , Geòrgia , Bodoni , Bitstream Cyberbit, Baskerville, Bookman Old Style.
    • Sans-serif. Ja explicada, per exemple: Arial , Verdana , Helvetica , Trebuchet , Gill Sans , Futura , Tahoma , Geneva, etc
    • Cursiu. Són lletres normalment inclinades i molt ornamentades que simulen l'escriptura manual. Per exemple: Zapf-Chancery , Comic Sans , Script MT, Catflish Script, Adobe Poetica, Monotype Corsiva , Brush Script, etc.
    • Fantasy. Lletres molt ornamentades que no estan pensades per al text normal d'un document, però sí que es podria utilitzar per als seus títols (però només per als que volem recarregar). Per exemple: Western , Impact , Copperplate , Artistik , Britannic Bold , Algerian, Herculanum, etc.
    • Monospaced. Lletres monoespaiades, cada lletra ocupa el mateix. Simulen escriptura amb màquina d'escriure antiga. Exemples: Courier , Lucida Console , Consoles , Monaco , Liberation Mono, Source Code Pro , Andale Mono, etc.

    Importar tipus de lletra

    @font-face

    Un dels problemes més importants sobre la tipografia és voler utilitzar un tipus de lletra sobre la qual, no hi ha seguretat que hi haja al dispositiu de la persona que està veient la pàgina HTML.

    CSS3 ha introduït l'ús d'aquesta directiva al codi CSS per precarregar el tipus de lletra necessari. Per això haurem d'utilitzar el fitxer que conté el tipus de lletra en si. El format possible per als fitxers amb el tipus de lletra és:

    • ttf. Lletres de tipus TrueType, les clàssiques de Windows. Utilitzades per Mozilla i Chrome/Safari (i altres).
    • otf. Lletres de tipus Open Type, són les que s'usen als sistemes Windows actuals. Està basada en el format anterior. Utilitzades per Mozilla i Chrome/Safari (i altres).
    • eot. Propietat de Microsoft (i per tant compatible amb els navegadors d'aquesta empresa)
    • woff. Web Open Font Format, format obert per a la web impulsat per Mozilla i compatible en quasi totes les versions dels ultims navegadors. Es va crear per evitar problemes de propietat dels formats anteriors.
    • svg. Format estàndard vectorial de gràfics, que permet també dissenyar tipus de lletres. Ho reconeixen alguns navegadors (especialment per al mòbil).

    Amb tants formats i incompatibilitats podem tindre el problema que la font escollida al final no es vegi correctament. Per això serveis com el disponible a la pàgina Font Squirrel ( http://www.fontsquirrel.com/tools/webfont-generator ) permet pujar la lletra en format Open Type (suposant que la font no tinga drets reservats o bé els hàgem pagat ) i ens descarrega tots els fitxers en els altres formats i fins i tot el codi que cal escriure al nostre full d'estils CSS per carregar la font correctament en qualsevol navegador compatible amb CSS3.

    Per carregar la font s'usa la instrucció @font-face a la qual se li incoporen diverses propietats entre les que estan almenys font-family.

    Per exemple, suposem que volem utilitzar el tipus de lletra Cantarell i disposem (tenint en compte que els tipus de lletra poden tindre drets exclusius d'ús, per la qual cosa cal assegurar-vos que ús és lliure o bé pagar els drets per al seu ús) de fitxer Artistik.ttf que conté el tipus de lletra en si. El codi perquè tots els paràgrafs la usen (amb la seguretat que sí que sortiria) és:

    • font-family. Per donar nom de família a la lletra que estem important. Aquest nom és el que després utilitzaran els selectors per triar la lletra que hem carregat (a l'exemple és el que utilitza p per donar format amb aquesta lletra als paràgrafs).
    • src. En què s'indica una o més URL (separades per comes) que apunten a fitxers de lletres que es carregaran a la pàgina perquè es veja correctament la lletra. El fet de donar diverses rutes amb fitxers de tipus de lletra és per proporcionar opcions, per en cas de no funcionar el primer fitxer carregar el segon i així successivament.

    Opcionalment podem utilitzar les propietats:

    • font-weight.
    • font-style.
    • font-stretch.

    Totes elles vistes anteriorment i que ara serveixen per indicar de quina mena és la font que s'està carregant; si és negreta, cursiva, etc.

    Exemple:

    
@font-face {
    font-family: cantarellbold;
	/* primera opció, tipus de lletra EOT, no admet utilitzar opcions */
    src: url( 'font/cantarell-bold-webfont.eot' );
	/* opcions en ordre de prioritat desitjada per importar els tipus de lletra */
    src :
     local( "Cantarell Bold" ),
     url( 'font/cantarell-bold-webfont.eot?#iefix' ) format( 'embedded-opentype' ),
     url( 'font/cantarell-bold-webfont.woff' ) format( 'woff' ),
     url( 'font/cantarell-bold-webfont.ttf' ) format( 'truetype' ),
     url( 'font/cantarell-bold-webfont.svg#cantarellbold' ) format( 'svg' );
    font-weight:  bold;
    font-style: normal;
}
p{
	font-family: cantarellbold;
	...
}


    A l'exemple anterior a la propietat src s'indiquen diverses rutes mitjançant la funció url i a cada una s'indica el format del fitxer mitjançant la funció format. Abans de fer servir les rutes, es cercarà la font Cantarell Bold a l'ordinador local gràcies a la funció local i així si l'usuari ja la té al sistema no caldrà carregar-la des del fitxer extern.

    Encara que aquesta tècnica és molt interessant perquè ens permet utilitzar fonts personals i no dependre que l'usuari o usuària té instal·lat al vostre ordinador, cal carregar l'arxiu de fonts i això pot portar cert temps, durant el qual no es veu cap text o es veu amb un format diferent, cosa que pot ser problemàtica. No cal abusar d'aquesta tècnica carregant massa fonts perquè si no la càrrega de la pàgina web seria molt lenta.

    Repetició del fons. background-repeat

    En principi, el fons es repeteix a totes les adreces fins a omplir l'element al qual s'aplica la imatge de fons. Però podem fer que la imatge es repeteixi en una sola direcció (horitzontal o vertical), i fins i tot fer que no es repetisca.

    Possibles valors de la propietat:

    • repeat. És el valor per defecte. la imatge es repeteix a totes les adreces (efecte mosaic), fins a omplir completament l'element.
    • repeat-x. La repetició de la imatge es fa només en horitzontal.
    • repeat-y. La repetició de la imatge es fa només en vertical.
    • no-repeat. La imatge no es repeteix apareixerà només una vegada

    Posició de la imatge. background-position

    Per defecte, la imatge es col·loca des del cantó superior esquerre de la pàgina (posició 0,0) i des d'ací es repeteix (si s'ha indicat repetició de la imatge amb la propietat anterior) o no.

    La posició des de la qual la imatge es col·loca inicialment es pot modificar. La manera de fer-ho és així:

    background-position: posicióHorizontal posicióVertical


    Podem indicar ambdós valors així:

    • posicióHorizontal. Es poden especificar les paraules: left (esquerra), right (dreta) o middle (centre). També es pot indicar una coordenada horitzontal concreta (per exemple 12px ) o relativa ( 5% ).
    • posicióHorizontal. Es poden especificar les paraules: top (a dalt), bottom (a baix) o center (centre). També es pot indicar una coordenada vertical concreta (per exemple 20px ) o relativa ( 15% ).

    Així per exemple el codi:

    background-position:  right bottom;


    La imatge de fons (si no hi ha repetició) es mostrarà a la part inferior dreta de l'element on es pose.

    Desplaçament del fons. background-attachment

    Quan es desplaça el contingut d'una pàgina web, el fons es mou amb la resta de la pàgina. Mitjançant aquesta propietat podrem fer que el fons quede fix mentre només la resta d'elements es mouen. Els possibles valors per a la propietat són:

    • scroll. Valor per defecte. El fons i el text es mouen junts quan lusuari es desplaça per la pàgina
    • fixed. El fons és fix i només es mou el text.

    Mida del fons. background-size

    Permet indicar la mida de la imatge de fons indicant la mida en horitzontal i en vertical. És una propietat de CSS3 que només funciona als navegadors més moderns.

    Exemple (Fa que la imatge de fons medisca 320px per 240px):

    background-size: 320px 240px;


    Una altra possibilitat és:

    background-size: 50% auto;


    La paraula auto fa que la dimensió en què apareix aquesta paraula (en l'exemple fa referència a l'alçada) es calcule automàticament.

    Més recentment (i per tant no reconegut per navegadors que no siguen prou moderns) a aquesta propietat li podem donar els valors:

    • cover. Fa que la imatge ompli el contenidor on es troba de forma responsive , és a dir, sense perdre la seua proporció d'alçada i alçada. Fa que la imatge siga el més gran possible sense perdre la proporció d'alçada per altura.
    • contain. Escala la imatge fent que cobrisca el 100% de l'amplada o de l'alçada depenent de la mida de la finestra de manera que quedaran buits al contenidor de la imatge ja que l'altra proporció no s'escala.

    Àrea de dibuix del fons. background-clip

    És una propietat de CSS3 que defineix des de quina zona es dibuixa el fons. Les seves possibilitats són:

    • border-box. Opció per defecte, el fons es dibuixa des d'on normalment hi ha la vora d'element al què estem dibuixant un fons.
    • padding -box. El fons es dibuixa a l'àrea de reompliment ( padding ) que és una zona interior a la vora.
    • content-box. El fons es dibuixa a la zona de contingut que és on es col·loca el text a l'element.

    Origen del fons. background-origin

    És una propietat CSS3 que encara no és molt compatible amb els navegadors. Té les mateixes possibilitats que l'anterior; però, en aquest cas, fa referència des de prenem l'origen de coordenades que ens servirà com a referència per posicionar les imatges de fons.

    propietat background

    Fixa en una sola propietat totes les propietats de fons. Sintaxi:

    background: background-color background-image
    background-repeat background-attachment
    background-position


    Exemple:

    background: maroon url( 'fondo1.gif' ) no-repeat fixed left bottom;

    Elements del format de caixa

    Se sol anomenar format de caixa (o format de quadre ) a la part de CSS encarregada del format referent al rectangle imaginari que envolta un element d'una pàgina HTML. Aquest rectangle imaginari conté completament l'element, així com la vora (si n'hi ha) i l'espai que es deixa fora del marge ( margin ) i l'espai que es deixa per dins de la vora ( padding ).


    L'espai que va des del contingut de l'element (a la il·lustració seria el text de color negre) fins a la vora és el que s'anomena espai de farciment o padding. Aquest espai queda pintat o emplenat amb el fons que s'hagi configurat.

    El marge és un espai exterior a la vora. En aquest espai no es veu la vora, és un espai transparent veurem en ell, el fons configurat al contenidor de l'element. És a dir si l'element és una etiqueta p que penja directament de body, a l'espai de marge veurem el fons de l'element body .

    Configuració de la vora

    propietats per a la vora

    La vora té tres propietats fonamentals: color, estil i gruix. A més, podrem configurar independentment cada vora (esquerra, dret, superior i inferior). Pel que disposem de totes possibilitats:

    • border-width. Amplada de la vora, indicada a la unitat desitjada. Per exemple 12px indicaria una vora de dotze píxels. També podem indicar aquests valors:
      • thin. Vora fina
      • medium. Vora de gruix mitjà
      • thick. Vora gruixuda

      border-width: thick thin thick 0px;

      Aquest codi aplica vores superior i inferior gruixudes, a la dreta fina i a l'esquerra no dibuixarà cap vora. No cal indicar totes les vores. Per exemple:

      border-width: thick thin;
/* superior i inferior gruixut, esquerre i dret fi */
border-width: thick thin 0px;
/* superior gruixut, esquerre i dret fi i inferior sense vora */


    • border-style. Modifica l'estil de la vora. Aquesta propietat afecta totes les vores (superior, dret, inferior i esquerre). Possibilitats:
      • solid. Vora sòlida
      • dashed. Vora ratllada
      • dotted. Puntejat
      • double. Doble
      • groove. Vora 3D amb efecte d'enfonsament
      • ridge. Vora 3D amb efecte d'elevació
      • inset.
        L'efecte depèn del color escollit
      • outset.
        L'efecte depèn del color escollit.
      • none

      Com passa amb border-width, podem indicar diversos estils alhora. Exemples:

      border-style: solid dashed dotted dotted;

border-style: solid dashed dotted;
/* superior sòlid, esquerre i dret ratllat i inferior puntejat */

    • border-left-width, border-left-color, border-left-style. Permet indicar respectivament l'amplada, el color i l'estil de la vora esquerra.
    • border-bottom-width, border-bottom-color, border-bottom-style. Permet indicar respectivament l'amplada, el color i l'estil de la vora inferior.
    • border-right-width, border-right-color, border-right-style. Permet indicar respectivament l'amplada, el color i l'estil de la vora dreta.
    • border-top. Permet indicar alhora les tres propietats per a la vora esquerra. L'ordre és: gruix, estil i color. Exemple: 
      border-top: 2px solid red;

      No estem obligats a emplenar les tres propietats podem emplenar només una o dues.

    • border-right. Permet indicar, alhora, les tres propietats de la vora dreta.
    • border-bottom. Permet indicar, alhora, les tres propietats de la vora inferior
    • border-left. Permet indicar, alhora, les tres propietats de la vora esquerra
    • border. Permet indicar el gruix, estil i color de les quatre vores alhora. S'usa igual que les anteriors però en aquest cas el que configurarem implica totes les vores.

    Gradients lineals

    Un gradient és un degradat que permet pintar en lloc d'un color únic, amb una variació que va d'un color a un altre. CSS3 permet l'ús d'una funció anomenada linear-gradient que permet utilitzar-se per exemple a la propietat background per establir un gradient.

    Sintaxi de la funció (estàndard):

    linear-gradient( direccion,primerColor posicion1,segonColor posicion2,… )


    El significat dels paràmetres és el següent:

    • direcció. És opcional i serveix per dir en quina direcció es dibuixarà el gradient. Possibilitats: indicar un angle (per exemple 45deg ) o una direcció a un cantó o costat de la pantalla usant la paraula to. Exemples:
      • 45deg. 45 graus d'inclinació
      • to top. DEgradat vertical
      • to left. Degradat horitzontal
      • to top left. Degradat en direcció a la cantonada superior esquerra
    • colors (primer, segon,..). Són els diferents colors del degradat, com més posem, més bandes tindrà.
    • posicions. Per a cada color podem indicar-ne la posició, normalment un percentatge. Si no es fa servir, els colors es reparteixen uniformement a l'àrea del degradat.

    Perfils

    Els perfils són vores que es posen a l'exterior dels elements de tipus caixa.

    A diferència de les vores, pròpiament dites, explicades als apartats anteriors, els perfils són línies que no mengen espai ni al farciment ni al marge. Els perfils són vores que es col·loquen a la posició en què normalment ocupa una vora però no ocupen un espai extra, com si la línia que dibuixen estiguera per baix del text. Les propietats dels perfils són:

    • outline-color. Color del perfil.
    • outline-width. Amplada del perfil
    • outline-style. Estil del perfil (admet les mateixes possibilitats que border-style).
    • outline. Resumeix les tres anteriors en una sola propietat, admet posar primer el color, després l'estil i després l'amplada.

    L'estàndard CSS va afegir aquestes propietats no per substituir les vores, sinó per ser aplicats a elements de formulari per tal que quedarà marcat l'element que actualment té el focus; és a dir, aquell element que rep les dades procedents del dispositiu dentrada de lusuari, normalment el teclat. S'utilitzen també per a depuració i per donar format CSS a esdeveniments de tipus hover, ja que la vora s'afegeix sense que es moguin els elements al seu voltant, cosa que provoca un efecte més elegant.

    Vitsualització d'elements

    Propietat display

    Maneres d'aparèixer un element en un navegador

    Els diferents elements HTML tenen una manera per defecte de presentar-se per pantalla. Així, per exemple, els elements p o h1 utilitzen una àrea rectangular, mentre que els elements strong o em es mostren seguits a la pantalla.

    La propietat que se n'encarrega és display, una propietat apareguda a CSS2 que permet establir la forma en què es mostra per pantalla un element. Està especialment indicada per a documents diferents a HTML (XML especialment) en què el seu contingut no té per defecte una manera de mostrar-se.

    En HTML permet modificar la manera com es mostra un element. Per exemple, en HTML els elements de tipus p (paràgrafs) s'entén que es mostraran com a paràgrafs independents, és a dir, amb cada etiqueta p comença un nou paràgraf o bloc. No aniran seguits dos elements de paràgraf consecutius.

    Principals valors de display:

    • none.

      L'element queda ocult (també els elements fills) i l'espai que ocupava l'element es retira. Desapareixerà fins i tot de l'arbre d'elements del document, cosa que impedirà, per exemple, l'accés a aquest element des de JavaScript.

    • inline.

      Els elements amb aquest display es mostren sense trencar la línia actual. Es mostren dins de la línia. Per tant podem col·locar dos elements inline seguits.

      Els elements inline ignoren els marges superior i inferior, però sí que reconeixen els farcits i els marges esquerre i dret, així com les vores i els colors i imatges de farciment. No fan cas de les propietats width, height.

      Exemples d'elements que són per defecte d'aquest tipus serien: span, small, strong, em, mark, abbr, kbd, etc.

    • block.

      Són elements que es mostren dins una caixa rectangular. Trenquen la línia actual i, per defecte, no admeten cap altre element al seu costat (independentment de la mida o l'alineació).

      Els elements de tipus block admeten manipular qualsevol propietat. Elements, per defecte, d'aquest tipus són: p, h1, div, section, article, nav, figure, ul, etc.

    • inline-block.

      L'element forma un bloc rectangular, però és interior al contenidor (no trenca la línia actual). Sí que admeten elements al seu costat. Admeten manipular les propietats width i height el que demostra la seua semblança amb el valor block.

      Exemples d'elements d'aquest tipus són: button, image, input, textarea, select, etc.

    Altres valors de display:

    • list-item. L'element es considera un ítem d'una llista (igual que l'etiqueta li). Per tant, els elements així marcats admeten les propietats: list-style-type, list-style-image, list-style-position i list-style .
    • run-in. L'element es comporta com a block o com a inline depenent del context.
    • table. Considereu que l'element conté una taula (com l'element table).
    • table-caption. Considereu a l'element un títol de taula (com passa amb caption).
    • table-cell. Considereu a l'element una cel·la d'una taula (com th o td ).
    • table-row. Considereu que l'element conté una fila d'una taula (com a tr).
    • table-row-group. Considereu a l'element un grup de files (com tbody).
    • table-column. Considereu a l'element una columna (com col ).
    • table-column-group. Considereu a l'element un grup de columnes (com colgroup).
    • table-footer-group. Considereu que l'element conté el peu d'una taula (com tfoot)
    • table-header-group. Considereu a l'element una capçalera de taula (com theader)
    • inline-table. Element inline interior a una taula.

    Valors de display per a maquetacions avançades.

    • flex. Permet establir contenidors de tipus caixa flexible que façaliten el disseny adaptable d'una pàgina. Només ho reconeixen les darreres versions de navegadors
    • inline-flex. Semblant a l'anterior, però l'element s'entén que es mostra a la mateixa línia que els seus elements adjacents.
    • grid. Element que conté una estructura de tipus grid.
    • inline-grid. Element que conté una estructura grid de fila.
    • subgrid. Element interior a un grid que també es comportarà com un grid.

    Tots aquests valors són molt moderns (alguns no funcionen a gairebé cap navegador dels existents a l'hora d'escriure aquestes línies).

    Introducció

    Una de les grans i històriques peticions dels dissenyadors web ha estat la possibilitat de poder col·locar contingut en una posició concreta de la pàgina, fent que els continguts no es vagen mostrant en l'ordre en què apareixen al codi, sinó en què decidim lliurement.

    Va ser la principal aportació de CSS2 i ha provocat que, en lloc de fer servir taules per definir la maquetació de la pàgina, ara siguen les capes les encarregades de fer-ho. Les capes són elements que permeten una distribució de continguts més lliure. L'etiqueta div sol ser l'encarregada de crear capes, encara que amb HTML5 són altres elements els que també s'usen per a aquesta tasca com a section o article , ja que div no és una etiqueta semàntica (no diu quin tipus de contingut posseeix, si és un article, un títol, un menú,…).

    Cal tindre en compte que la posició d'un element també estarà influït pel valor de la propietat display, com es va explicar al punt anterior.

    Flotació

    La propietat float és molt útil per a continguts multimèdia com a imatges o taules. Permet indicar com es comporta el contingut dels elements adjacents sobre l'actual.

    Normalment tots els elements no floten. És a dir, quan col·loquem, per exemple, una imatge, el contingut que va a continuació apareix a la seua línia base, deixant un buit poc estètic.

    • left. L'element flota a l'esquerra. El contingut immediatament següent el contorneja a la dreta, sempre que càpiga
    • right. L'element flota a la dreta. El contingut immediatament següent us contorneja a l'esquerra.
    • none. L'element no flota (valor per defecte). En el cas del text, la primera línia aniria seguida a l'element, la resta cauria per sota.

    Mida de l'element

    Dues propietats permeten establir la mida d'un element:

    • width.

      Amplada de l'element. El seu valor per defecte és auto, la qual cosa implica que l'element es farà prou ample perquè en càpiga el contingut. D'altra manera indicarem la seua mida utilitzant la unitat de mesura desitjada ( px , em , % , etc.).

      En el cas de les unitats relatives, es pren com a referència allò que mesura el contenidor; si indiquem un 50% a un element que està dins d'un element div , prendrà la meitat de l'amplada d'aquest div, si el nostre element penja directament de body , prendrà la meitat de l'amplada de la pàgina.

    • height.

      Alçada de l'element. Teniu les mateixes possibilitats que l'element anterior.

    • max-width.

      S'utilitza quan la mesura de l'amplada d'un element s'indica relativament, per exemple amb percentatges. Aquesta propietat indica una mida màxima per a l'element.

      És a dir, si hem indicat width: 50%; i a més max-width:750px; aleshores l'element mesurarà d'amplada la meitat del contenidor. Si per exemple el contenidor fa 2000 píxels, el nostre element mesuraria 1000 píxels aplicant el 50%; però com hem indicat la propietat max-width:750px , mesurarà exactament 750 píxels.

      Aquesta propietat és molt interessant per aplicar dissenys de tipus adaptable.

    • min-width.

      La idea és la mateixa que a la propietat anterior, però ara s'indica una mida mínima que tindrà prioritat sobre la mida relativa de la propietat width.

    • max-height.

      La mateixa idea s'aplica a l'alçada de l'element. Indiqueu l'alçada màxima que podreu prendre l'element.

    • min-height.

      Alçada mínima de l'element.

    El valor per a aquest posicionament és fixed. Amb aquest posicionament, l'element flota lliurement per la finestra i abandona el buit que, pel seu ordre natural, li correspondria. És a dir, pren com a referència la finestra del document de manera que el cantó superior esquerre serà la coordenada 0,0.

    Coordenada z

    La pantalla és un mitjà en dues dimensions i, per tant, la posició dels elements s'indica amb les coordenades x i y. No obstant això, es pot simular la tercera dimensió (la profunditat) mitjançant una coordenada fictícia, z.

    Realment, el que aporta és la possibilitat que els elements se solapen i puguem decidir quin element està per sobre, de manera que qui tinga la coordenada z més alta diríem que és més a prop de l'espectador i per tant té prioritat; és a dir, quedarà sobre els altres.

    És el mateix efecte que aconsegueixen els sistemes operatius gràfics com Windows o OS X quan tenim dues finestres solapades; una tapa l'altra entenent que està per davant .

    La propietat que permet controlar la coordenada z és z-index a la qual se li atorga un número enter de manera que com més gran sigui, més quedarà per davant dels altres elements. El text normal (d'elements que no tingan marcat un z-index) és al nivell zero; amb la qual cosa, si indiquem un nivell negatiu a la propietat z-index d'un element, entendrem que darrere del contingut normal (no es veurà l'element, quedarà tapat).

    Visibilitat dels continguts

    Desbordament

    El contingut de les capes pot ocupar més que les dimensions que li hem donat. En aquest cas podem establir com comportar-nos respecte al contingut

    Overflow , desbordament

    Es tracta d'una propietat que permet decidir la política que ha d'adoptar el navegador quan un contingut supera la mida prevista del contenidor. Possibles valors:

    • visible. El contingut es mostra encara que supere la mida prevista.

    Transformacions 3D

    En aquest cas, cal tindre en compte que permeten distingir l'eix sobre el qual s'apliquen. Les possibles funcions que podem utilitzar a la propietat transform per aconseguir transformacions 3D són:

    • translate3D( x , y , x ). Trasllada l'element poden indicar desplaçaments als 3 eixos alhora.
    • translateX( x ). Trasllada l'element a l'eix X.
    • translateY( y ). Trasllada l'element a l'eix Y.
    • translateZ( y ). Trasllada l'element a l'eix Z.
    • scale3D( x , y , x ). Escala l'element poden diferenciar escales en les tres dimensions de l'element
    • escaleX( X ). Escala l'element en horitzontal
    • escaleY( Y ). Escala l'element en vertical
    • escaleZ( Z ). Escala l'element a l'eix Z
    • rotate3D( x , y , x ). Trenca l'element poden indicar una rotació diferent a cada eix
    • rotateX( X ). Trenca l'element usant l'eix X de coordenades. Un element normal rotat més de 90 graus farà un efecte mirall. Rotant 90 graus exactes, l'element queda invisible en quedar absolutament horitzontal a la nostra vista.
    • rotateY( Y ). Trenca l'element usant l'eix I de coordenades. Com a la rotació anterior, un element normal rotat més de 90 graus, farà un efecte mirall. També just amb 90 graus l'element queda invisible en ser absolutament vertical a la nostra vista.
    • escaleZ( Z ). Escala l'element a l'eix Z, és la rotació 2D normal per tant.
    • perspective(n). Defineix un punt de perspectiva per a les transformacions 3D. És equivalent a utilitzar la propietat perspective: n.

    Hi ha altres propietats relacionades amb les transformacions 3D:

    • transform-style. Si indiquem a aquesta propietat el valor preserve-3d, si aquest objecte conté objectes transformats en 3D, aquestes transformacions es mantindran. L'altra opció és indicar el valor flat.
    • transform-origin. Permet traslladar l'objecte transformat a una altra posició, indicant valors d'eix X i Y.
      Exemple transform-origin: 200px 100px;
    • perspective . Posició de la perspectiva de la persona que observa aquesta escena.
    • perspective-origin. Permet indicar on és l'eix de coordenades X, Y. Per defecte val 50% 50% ( just al centre). Admet indicar les paraules left , center o right per a l'eix horitzontal i top , center o bottom per al vertical

    Transicions

    <body> 
        <div>   </div>

    Mitjançant les transicions podem fer que aquest canvi es faça progressivament (d'altra manera, el canvi apareix de cop). Les propietats relacionades amb transicions són:

    • transition-property. Propietat que canviarem. Les propietats que es poden indicar són width , height , background-color ,.. És a dir, en principi qualsevol propietat CSS. Però com és lògic els navegadors només poden fer la transició sobre algunes. En indicar sobre quina propietat realitzem la transició només podem indicar-ne una. Però podem indicar all (valor per defecte) perquè faça la transició sobre totes les propietats alhora.
    • transition-duration. Durada de la transició, es mesura en temps.

      Exemple:

      transition-duration: 2s;

    • transition-timing-function. Funció de transició que es farà servir per mostrar la transició pot prendre un d'aquests valors:
      • linear. És la que es fa servir per defecte, la transició té sempre la mateixa velocitat.
      • ease. La transició frena la velocitat al final.
      • ease-out. La velocitat de la transició va a poc a poc disminuint.
      • ease-in. La velocitat de la transició comença lenta i després va accelerant.
      • ease-in-out. La velocitat comença frenada, accelera i torna a frenar al final.
      • cubic-bezier. Permet indicar 4 valors per realitzar una funció de temps que seguiu la funció cúbica de bezier. Amb coneixements sobre aquest tema, es poden fer efectes de transició i temps a mida.
    • transition-delay. Temps que triga a començar a fer l'animació. És un temps despera inicial.
    • transition. Resumeix les funcions anteriors. El més normal és utilitzar aquesta propietat en lloc de les anteriors. La seua sintaxi és: 
      transition: propietat temps funcióDeTemps retard;

    No cal omplir tots els valors. És a dir, podem indicar només la propietat i el temps (la resta prendran els valors per defecte)

    Cal tindre en compte que només els navegadors recents suporten aquesta propietat, abans tots ells requerien l'ús de prefixos.

    Animacions

    Les animacions són semblants a les transicions amb la diferència que tenen la possibilitat que s'executin directament, sense fer cap esdeveniment.

    Són més potents (i més complexes de crear) que les transicions i tenen una gran connexió amb el llenguatge JavaScript.

    Les animacions requereixen primer un bloc @keyframes (per a navegadors webkit cal utilitzar a més la variant @-webkit-keyframes) que és l'encarregat d'establir les propietats de l'animació i de donar-hi nom. Dins aquest bloc s'estableixen dos elements:

    • from. Amb les propietats inicials de l'element a què s'aplica l'animació.
    • to. Amb les propietats finals.

    Exemple:

    @keyframes ani1{
	from {
		left: 20px;
		background-color: red;
	}
	to {
		left: 220px;
		background-color: blue;
	}
}


    També podem establir indicar en lloc d'origen i final passos en percentatge, per exemple:

    @keyframes ani1{
	from {
		left: 20px;
		background-color: red;
	}
    50%{
		background-color: green;
    }
	to {
		left: 220px;
		background-color: blue;
	}
}


    En aquesta animació, el color de fons es va canviant a verd abans de convertir-se en blau.

    Les propietats que poden ser animades són les mateixes que les comentades a l'apartat dedicat a les transicions. En realitat transicions i animacions operen de manera molt semblant.

    Les propietats relacionades amb les animacions són:

    • animation-name. Permet utilitzar l'animació el nom del qual es va indicar en una secció @keyframes.
    • animation-duration. Temps ( 2s per exemple) que durarà l'animació.
    • animation-timing-function. Funció de maneig de l'animació. És una de les vistes a l'apartat anterior per a la propietat transition-timing-function.
    • animation-delay. Temps que tardarà a iniciar-se l'animació des que es llance
    • animation-iteration-count. Per defecte, pren el valor infinite que significa que l'animació es repeteix indefinidament. Podeu agafar un número que, llavors, indicarà el nombre exacte de vegades que es repetirà l'animació.
    • animation-direction. Pot valer normal o alternate. En el primer cas, quan l'animació s'acaba torna a començar des del primer fotograma. En el segon cas en acabar fa el moviment contrari fins a arribar al primer fotograma (és com fer un play i un play revers en un vídeo).
      És a dir, el valor alternate en una animació que mostre una capa movent-se d'esquerra a dreta, farà que en acabar l'animació, ara continue de dreta a esquerra fent el moviment contrari.

    Com sol passar en CSS, hi ha una sola propietat que permet resumir totes les anteriors. Per això es fa servir la propietat animation (en navegadors Webkit és -webkit-animation ), que té aquesta sintaxi:

    animation: nom durada funcióDeTemps retard comptador direcció;
