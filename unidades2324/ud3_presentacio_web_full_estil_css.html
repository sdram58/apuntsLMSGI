Introducció als fulls d'estil

Concepte de fulls d'estil

Els fulls d'estil són elements agregats al llenguatge Html que tindran en compte la presentació del document o de l'aplicació Web.


El mateix contingut html es pot presentar de manera diferent segons el full d'estil utilitzada. En aquesta web (css zen garden) es pot vore clarament com canvia una pàgina d'una altra només canviant l'estil

Ús de les fulles d'estil

  • Obligatòries en HTML5: Separació entre contingut i presentació.
  • Simplificació del codi
  • Manteniment del lloc web més senzill: Els fulls d'estil permeten, modificant un únic fitxer, actualitzar la presentació gràfica del conjunt de pàgines d'un lloc complet
  • Una via cap a l'accessibilitat: Els fulls d'estil ofereixen, per exemple, la possibilitat de visualitzar una pàgina de text amb caràcters més grans i, per tant, més llegibles.

Breu Història

  • 1991 Tim Berners- Lee inventa la web, el llenguatge HTML es concep per al intercanvi dinformació entre els científics de tot el món.
  • 1994. Sorgeix la idea de proporcionar estil a les pàgines web
  • 1995. El W3C (World Wide Web Consortium) va començar a funcionar i publica el primer esborrany sobre els fulls d'estil.
  • 1996. L'esborrany es converteix en una recomanació oficial amb el nom "Fulls d'estil en cascada, nivell 1" o CSS1
  • 1998. Es publica una extensió de la recomanació amb el nom CSS2
  • 2006. Es publiquen algunes correccions i extensions baix el nom CSS2.1.
  • 2009. Es va començar a parlar de les fulles d'estil CSS3.

Validador de les fulles d'estil

Igual que per a HTML5, hi ha un validador per als fulls d'estil.

El validador en línia del W3C per als fulls CSS (http://jigsaw.w3.org/css-validator/) és una eina de gran valor amb una qualitat reconeguda unànimement. Permet lliurar pàgines perfectes, respectant escrupolosament les normes dictades pel W3C.


El validador CSS només comprova els fulls d'estil. D'aquesta manera, només cal introduir el fitxer de fulls d'estil extern o el contingut de l'etiqueta <style> per a les fulls d'estil interns del document.

Declaració d'estils i selectors bàsics

La declaració d'un estil es fa mitjançant el binomi propietat: valor: Exemple:

background-color: red;


Selectors d'etiqueta.

La sintaxi d'un selector d'etiqueta és el nom de l'etiqueta seguit de la declaració d'estil entre claus.
selector { declaració d'estil }

Exemple

div { background-color: red;}


Podem aplicar el mateix estil a diferents selectors d'etiqueta separant-los amb comes. Exemple:

h1, h2, h3 {declaració d'estils}


Selectors de classe

El dissenyador defineix els seus propis selectors.
.nom_de_la_classe { declaració(ns) d'estil}

Exemple

.roig { background-color: red;}

La crida a la classe es realitza de la següent manera:

<etiqueta class="nom_de_la_classe">

Exemple

<div class="roig"> ... </div>

Podem definir una classe únicament per a una etiqueta, per exemple

blockquote.roig { background-color: red;}

Amb aquesta definició de classe, només la podrem aplicar a l'etiqueta seleccionada, en aquest cas blockquote.

El primer caràcter de la classe no pot ser un número, un guió o un caràcter de subratllat.

Selectors d'identificador

L'identificador ha de ser únic per a cada element de la pàgina, per tant, el selector d'identificador s'invocarà una sola vegada al document. La definició d'un selector id és:

#nom_del_identificador {declaració d'estil}

Exemple

#roig { background-color: red;}

La cridada es fa mitjançant:

<etiqueta id="nom_d'identificador">

Exemple:

<div id="roig"> ... </div>


Comentaris

En un full d'estil CSS, un comentari comença amb els caràcters d'obertura barra obliqua i asterisc (és a dir /*) i acaba amb els caràcters asterisc i barra obliqua (és a dir */)

Exemple

/* Això és un comentari */

Unitats de mesura

Les fulles d'estil CSS permeten utilitzar nombroses unitats de mesura, bé en polzades (inxes), en centímetres, en mil·límetres, en punts, en piques, en píxels o en percentatge.

També es distingeix entre valors relatius, que poden variar segons l'ordinador utilitzat, i valors absoluts, que permaneixen constants siga quin siga el dispositiu o el programari utilitzat.

Notacions de colors

Els fulls CSS proporcionen múltiples notacions per declarar un color.

  • La notació hexadecimal clàssica, és a dir #ffcc00:. Defineix el color, o més aviat els tres components del color. roig (r de red), verd (g de green) i blau (b de blue), mitjançant una notació hexadecimal de tipus #RRGGBB.
    • Notació amb alpha: #RRGGBBAA s'hi afegeixen un parell més al final que fa referència al canal alpha
    • Notació curta: podem escriure-ho usant un sol digit per color/alpha. El resultat serà que per a cada color es duplica la lletra. Evidentment és menys precís. #FC0 será igual a #FFCC00
  • La notació decimal, per exemple, color: rgb(0, 0, 255). El codi RGB del color no es codifica segons el valor hexadecimal sinó per un número comprès entre 0 i 255.
    • rgba(R, G, B, A): s'afegiex un quart paràmetre que és el alpha. El grau d'opacitat. Aquest pot estar en percentatge o decimal de 0 a 1, sent 0 no es veu, completament traparent i 1 completament opac.
  • La notació en percentatge:, per exemple, color: rgb(25%, 50%, 0%). El valor 0% significa l'absència del component esmentat i 100% el valor màxim.
  • Paraules clau: per exemple, color: red.
  • La notació HSL (Hue Saturation Luminance), al valencià Tonalitat: Saturació, Lluminositat. La notació HSL consisteix en tres valors.
    • El primer s'expressa en graus, de 0° a 359° (el símbol de grau º no apareix a la notació). Es correspon amb un color de la roda cromàtica: roig (0°), groc (60°) , verd (120°), cian (180°), blau (240°) i magenta (300°)
    • El segon i el tercer valor s'expressen en percentatge i descriuen respectivament la saturació i la lluminositat. Per exemple, color: hsl(0, 100%, 50%) per al roig.

CSS integrades en un element HTML5

A HTML és possible afegir directament al codi una declaració d'estil en una etiqueta determinada.

<h1 style="color: red;">Título de nivel 1</h1>

És convenient evitar aquesta manera de procedir en virtut del principi de separació entre contingut i presentació. Això no obstant, pot resultar útil per donar prioritat a una propietat d'estil respecte a la que s'haja especificat a la declaració interna o externa.

<!DOCTYPE html>
<html lang="ca">
    <head>
        <title>Html5</title>
        <meta charset="UTF-8">
    </head>
    <body>
        <h1 style="font-style: italic;">Títol 1</h1>
        <h1>Títol 2</h1>
    </body>
</html>

i quedaria de la següent manera:

CSS internes al document HTML5.

L'ús principal dels fulls destil CSS consisteix a determinar un estil que s'aplique a la totalitat d'una pàgina HTML5.

El codi dels fulls d'estil s'agrupa a la capçalera del document, entre les etiquetes <head> i </head>.

La sintaxi es presenta de la següent manera:

<head>
    <style type="text/css"> 

        h1 { color: red;}

    </style>

</head>

CSS externes al document HTML5.

Les declaracions d'estil es poden agrupar en un fitxer extern (diferent) al fitxer Html.

Aquesta manera de fer respecta millor el principi de separació entre contingut i presentació.

Aquesta tècnica empra dos arxius:

  • El full d'estil extern: Un fitxer que conté la declaració d'estil. Aquest fitxer té l'extensió .css.
  • El document Html5: Entre les etiquetes <head> i </head>, s'afegeix un enllaç al full d'estil en qüestió. 
    <link rel="stylesheet" type="text/css" href="exemple.css">

    on:
    • link indica al navegador que el que segueix és un enllaç.
    • rel="stylesheet" necessita que aquest enllaç sigui relatiu a un full d'estil.
    • href="exemple.css" és l'escriptura clàssica d'un enllaç a Html.
    • L'enllaç pot ser absolut (començant per http://...) o relatiu.
    • Res no impedeix incloure diverses etiquetes <link> cap a fulls d'estil externs diferents.
    • Res no impedeix utilitzar, a més, un full d'estil intern.

Exemple:

<head>
    <title>Html5</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" type="text/css" href="exemple.css">
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>

CSS importada (@import).

Una altra forma d'usar els fulls d'estil externs consisteix a fer servir l'ordre @import.

<style type="text/css"> 
    @import url(exemple.css);
</style>


@import es podrà utilitzar no només per invocar un full d'estil extern en un document Html sinó també per importar un altre full d'estil al full d'estil extern.

<!DOCTYPE html>
<html lang="ca">
<head>
    <title>Html5</title>
    <meta charset="UTF-8">
    <style type="text/css">
        @import url(exemple.css);
    </style>
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>


Les etiquetes <style> ... </style> són necessàries.

Noció de cascada

Als apartats anteriors hem constatat com és possible tenir diverses definicions d'estil, bé en línia, interna(es) o externa(es). n cas que hi haja una competència entre diversos elements d'estil, intervé la noció de cascada (la paraula Cascading a Cascading Style Sheets) o ordre de prioritat.

El navegador pren primer en consideració les especificacions dels fulls d'estil externs (amb l'extensió css), a continuació els fulls d'estil interns (aquells situats a l'interior de les etiquetes <head>) i, tot seguit, els fulls d'estil en línia (aquelles lligades a un element Html5).

L'ordre creixent de prioritat (de menor a major) és:

  1. Propietats per defecte del navegador.
  2. Fulles d'estil externes.
  3. Fulles d'estil internes.
  4. Fulls destil en línia.

La regla de prioritat, per visualitzar el document al navegador, consisteix en utilitzar el full d'estil més proper a al element.

Exemple:

L'arxiu d'estil extern estil1.css es:

h1 { text-decoration: underline;}

<!DOCTYPE html>
<html lang="ca">
<head>
    <title>Html5</title>
    <meta charset="UTF-8">
        <link rel="stylesheet" type="text/css" href="estil1.css">
    <style type="text/css">
        h1 { text-decoration: overline;}
    </style>
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>
</html>

Donant com a resultat:

Títol de nivell 1

Es pot donar prioritat a l'estil extern, és a dir a l'estil de subratllat, mitjançant el valor !important.

El fitxer d'estil extern (estil2.css) és:

h1 { text-decoration: underline !important;}

El document Html5:

<!DOCTYPE html>
<html lang="ca">
<head>
    <title>Html5</title>
    <meta charset="UTF-8">
    <link rel="stylesheet" type="text/css" href="estil2.css">
    <style type="text/css">
        h1 { text-decoration: overline;}
    </style>
</head>
<body>
    <h1>Títol de nivell 1</h1>
</body>
</html>

Donant com a resultat:

Títol de nivell 1

Noció d'herència

Si tenim diverses etiquetes anidades amb diferents estils


Els estils s'hereten:

  • L'estil 2 es caracteritza pel propi estil més el de l'estil 1, que ha heretat.
  • L'estil 3 es caracteritza pel propi estil més el dels estils 1 i 2, que ha heretat.
  • Seguint la mateixa lògica podem utilitzar els termes "pare" i "fill".
  • L'estil 1 seria el pare de l'estil 2. De la mateixa manera, l'estil 2 i l'estil 1 serien pares de l'estil 3. O bé l'estil 2 és fill de l'estil 1 i l 'estil 3 és fill dels estils 2 i 1 .

Selectors

Selector universal

Permet aplicar un estil a totes les etiquetes del document. Això ho aconsegueix l'asterisc. Exemple:

* {
    color:black;
}

No se sol utilitzar així ja que és massa indiscriminada. Però sí que s'utilitza per a elements d'aquest tipus:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        table *{
            color:red;
        }
    </style>
</head>
<body>
    <p>Ix normal</p>
    <table>
        <tr>
            <th>Ix roig</th>
        </tr>
        <tr>
            <td>Ix roig</td>
        </tr>
    </table>
</body>
</html>


Permet pintar en roig el text que estiga dins una taula; no importarà que etiqueta siga la que està dins de la taula. Un altre exemple:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p *{ color:red;}
    </style>
</head>
<body>
    <p>Ix normal <span>Ix roig</span></p>
</body>
</html>


Ara ix de color roig el text dins de l'etiqueta span.

Selectors per atributs

Permet aplicar estils a un element quan aquest té un valor determinat sobre un atribut. Per fer-ho, s'indica l'atribut entre claudàtors [], seguit del signe d'igualtat i el valor entre cometes. Exemple:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p[lang="en"]{
            font-style: italic;
        }
    </style>
 </head>
 <body>
     <p lang="es">text que ix normalment</p>
     <p lang="en">text que ix en cursiva</p>
 </body>
 </html>

Es pot barrejar aquest tipus de definicions amb classes o definicions per identificador:

p.classe1[lang="en"]{
    font-style: italic;
}


En aquest cas, l'estil definit s'aplica a paràgrafs de classe 1 que estiguen marcats amb el valor a l'atribut lang (és a dir, que estiguen en anglès).

Fins i tot es pot utilitzar més d'un atribut:

p[lang="en"][spellcheck="true"]{
    font-style: italic;
}

En aquest cas s'aplica l'estil per als elements de tipus p que utilitzen els atributs lang i spellcheck amb els valors indicats.

Fins i tot podem utilitzar diversos valors per al mateix atribut:

p[lang="en"][lang="fr"]{
    font-style: italic;
}


Aplica la cursiva a paràgrafs el llenguatge dels quals sigui anglès o francès.

També podem indicar l'estil simplement per als elements que usen l'atribut independentment del valor:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p[lang]{
            font-style: italic;
        }
    </style>
</head>
<body>
    <p>Aquest text es mostra normalment</p>
    <p lang="en">This text is shown with italics (es mostra amb cursiva)</p>
    <p lang="ca">Aquest text també es mostra en cursiva</p>
</body>
</html>

D'altra banda, gràcies a CSS3 disposem d'aquestes possibilitats:

sintaxi

significat

element[atribut~="valor"]

Elements que usen l'atribut indicat que continguen el valor encara que separat d'altres valors per espais

element[atribut$="valor"]

Elements que utilitzen l'atribut i el contingut del qual finalitzi amb el valor indicat

element[atribut^="valor"]

Elements que utilitzen l'atribut i el contingut dels quals comenci amb el valor indicat

element[atribut|="valor"]

Elements que utilitzen l'atribut, el contingut del qual comenci amb el valor indicat ia més aquest valor sigui una paraula

element[atribut*="valor"]

Elements que utilitzen l'atribut indicat i continguen (arreu) l'atribut indicat

Exemple:

<!doctype html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
    p[lang*="ca"]{
    font-style: italic;
    }
    </style>
</head>
<body>
  <p lang="es-AR">Aquest text es mostra en cursiva</p>
  <p lang="ca-ES">Aquest text també es mostra a cursiva</p>
  <p lang="ca">Fins i tot aquest també</p>
  <p lang="en">Aquest no</p>
</body>
</html>

Altres selectors jeràrquics

sintaxis

significado

element1 ~ element2

S'aplica al element2 quan és germà del element1 i aquest el precedeix, encara que no siga immediatament.

element:empty

S'aplica quan l'element és buit

element:nth-child(número)

S'aplica a l'element indicat quan sigui el fill amb el número indicat (per exemple, número seria 3, per al tercer fill).

Es poden fer servir expressions més complexes mitjançant l'ús de la variable n per aconseguir fórmules més complexes (la n el navegador la substituirà automàticament per cada número sencer per calcular quines files se seleccionen)

Es permet també utilitza les paraules clau odd (imparell) i even (parell)

Exemples dús:

  • tr:nth-child(3). Selecciona la tercera fila d'una taula
  • tr:nth-child(odd). Selecciona les files senars
  • td:nth-child(even). Als parells
  • tr:nth-child(2n+1). Selecciona les files 1,3,5,... (les senars)
  • tr:nth-child(3n+1). Selecciona les files 1, 4, 7, 10, 13... (de tres a tres)

element:nth-last-child(n)

Funciona igual que l'anterior però compta els elements fills a partir de l'últim (és a dir, l'ordre del darrere cap endavant).

element:first-child

S'aplica a l'element quan és el primer fill

element:last-child

S'aplica a l'element quan és el darrer fill

element:only-child

S'aplica quan l'element és l'únic fill

element:first-of-type

Primer descendiente de su tipo

element:last-of-type

Últim descendent del seu fill

element:nth-of-type(n)

Funciona com nth-child, però fa referència al número de fill d'aquest tipus. No compte fills, sinó fills del tipus indicat.

Exemple:

<head>
    <style>
        article p:nth-of-type(2){
            color:red;
        }
    </style>
</head>
<body>
    <article>
        <h1>Primer fill, no s'acoloreix</h1>
        <p>Segon fill, primer de tipus p. No s'acoloreix</p>
        <p>Tercer fill, segon de tipus p. ES COL·LOREJA</p>
        <p>Quart fill, tercer de tipus p. No s'acoloreix</p>
    </article>
</body>

element:nth-last-of-type(n)

Com l'anterior però conta n des del final.

element:only-of-type

S'aplica quan l'element és l'únic fill d'aquest tipus

Pseudoclasses

Bàsiques

Les pseudoclases permeten associar estils a un selector quan li passa una determinada circumstància. Inicialment les pseudoclasses s'aplicaven als enllaços (element a), però ara s'apliquen a qualsevol element. Les pseudoclasses clàssiques són:

  • :link. S'aplica als enllaços no visitats
  • :visited. Enllaços visitats
  • :activeu. Enllaços actius (aquells sobre els quals fem clic)
  • :hover. S'aplica quan el ratolí passa per sobre de l'element (seguiu un enllaç o no)

Exemple: 

a{
    text-decoration: none;
}
a:visited{
    color: gray;
}
a:hover{
    text-decoration: underline;
}

A l'exemple els enllaços als quals s'aplique aquest full d'estil sortiran sense subratllar, si estan visitats sortiran de color gris i si s'hi acosta el ratolí serà quan es mostrin subratllats.

Aquestes pseudoclases permeten un cert dinamisme a la pàgina HTML. Actualment, a més, és possible utilitzar-les en altres elements diferents de l'etiqueta a (com p, div, etc.), cosa que dóna enormes possibilitats.

Més pseudoclasses

CSS3 ha incorporat moltes més pseudoclasses:

pseudoclase

selecciona...

:focus

Quan l'element obté el focus, és a dir, quan l'element ha capturat l'entrada del teclat. Molt útil en formularis.

:lang(codi)

S'aplica quan l'element estiga marcat amb el llenguatge indicat pel vostre codi d'idioma a l'atribut lang (es per a espanyol, en per a anglès,…)

:enabled

Quan l'element està habilitat (útil als formularis)

:disabled

Quan l'element està deshabilitat (útil als formularis)

:checked

En controls de formulari de tipus radio o checkbox, quan l'element passa a estar activat.

:in-range

En controls de formulari (de tipus input) quan el valor que contenen està dins de rang. Això significa que els atributs min i max (que només s'apliquen a alguns tipus de controls input) han delimitat valors el contingut del control compleix.

No funciona a Internet Explorer.

:out-of-range

A l'inrevés de l'anterior. Aplica el contingut CSS a controls els valors dels quals estan fora de rang.

:target

S'aplica quan l'element a què fa referència ha estat destinació d'un enllaç. Exemple:

#intro:target{
    color:blue;
}

...
<h1 id="intro">Introducció</h1>
...
<a href="#intro">Anar a la introducció</a>


Quan fem clic sobre l'enllaç anar a la introducció, el títol Introducció es posarà de color blau.

:valid

Controls de formulari que contenen valors vàlids (ho seran o no depenent del tipus d'element). Per exemple, no serà vàlid un text en un quadre input de tipus numèric (number).

:invalid

Invers a l'anterior. És a dir, selecciona controls els valors dels quals no siguin vàlids.

:read-only

Controls de formulari que estan en mode de només lectura (a causa de l'atribut HTMl readonly). No funciona a Internet Explorer i a Mozilla Firefox només si es fa servir amb el prefix -moz-

:read-write

Invers a l'anterior. Controls que es puguen llegir i escriure.

:required

Controls de formulari que estan marcats (gràcies a l'atribut HTMl required) per ser emplenats obligatòriament.

:optional

Contrari a l'anterior. Controls que no estan obligats a ser emplenats.

:default

Selecciona els elements d'un formulari que són les opcions marcades per defecte. Per exemple, el botó de ràdio que actualment estiga activat. No funciona a Internet Explorer.

:first-line

La primera línia de l'element.

:first-letter

La primera lletra de l'element.

:before

Contingut anterior a l'element. Sempre s'utilitza amb la propietat content per afegir contingut a l'element. Exemple:

p:before{
  content:url('fletxa.gif');
}


Col·loca la imatge d'una fletxa davant del paràgraf

:after

Per indicar contingut després de l'element.

:lang(codiLlenguatge)

Seleccioneu els elements que hagin utilitzat l'atribut lang amb el codi indicat. És equivalent a [lang="codiLlenguatge"].

:not(selector)

S'aplica quan l'element no compleix el tipus indicat pel selector dins la paraula not. Exemple:

div >:not(p){
    color:red;
}


El color roig s'aplica al contingut dels elements div llevat que estiguen dins d'elements p.

Un altre exemple:

p:not(:first-of-type){
    color:green;
}


S'aplica a tots els paràgrafs excepte el primer de cada grup de germans.

:root

Només té utilitat quan s'aplica CSS sobre documents XML. Aplica l'estil a l'ítem quan és arrel del document. En un document HTML, l'arrel és sempre l'element html, per la qual cosa no té utilitat.

:dir(direccion)

Encara que és estàndard, no ho suporta encara gairebé cap navegador. Selecciona sobre la base de l'adreça on s'escriu el text. Aquesta adreça pot prendre el valor ltr (esquerra a dreta) o rtl (dreta a esquerra)

:fullscreen

Selector interessant, però experimental que s'aplica als elements que es mostren quan la pàgina HTML està en mode de pantalla completa. Tots els navegadors actuals la suporten però si es fa servir el prefix del navegador (per exemple -moz-fullscreen). Només el navegador Edge de Microsoft la reconeix sense prefix.

:left

S'aplica quan la pàgina s'imprimeix. Selecciona la pàgina esquerra (en impressió a doble cara).

:right

S'aplica quan la pàgina s'imprimeix. Selecciona la pàgina dreta (en impressió a doble cara).

Les pseudoclasses es poden combinar, de manera que es poden construir coses com:

input:focus:hover {
    background-color: yellow;
}

Es posarà color de fons groc, quan el quadre de tipus input tingui el focus i el ratolí passe per damunt. O coses més complexes com:

input.classe1:focus:hover[type="password"] ​​{
    background-color: yellow;
}

En aquest cas, s'aplica el fons groc quan el cursor estiga damunt d'un control de text de tipus contrasenya que, a més, tinga el focus i siga de classe1.

pseudoelements

Es tracta d'una millora de CSS3 que permet aplicar estils a elements, encara que aquests no existeixin a l'arbre d'elements del document. La raó perquè un element no estiga a l'arbre és que no existeix al codi fins que s'executi, per exemple, una determinada sentència en llenguatge JavaScript.

Els pseudoelements utilitzen els símbols :: en lloc de simplement : com passa amb les pseudoclases.

pseudoelement

significat

::first-line

Primera línia de l'element

::first-letter

Primera lletra de l'element

::after

S'aplica per col·locar codi després de l'element

::before

Se aplica para colocar código antes del elemento

::selection

S'aplica al text seleccionat de l'element. En el cas dels navegadors de Mozilla, cal utilitzar el prefix -moz-

::placeholder

Pseudoclasse experimental (només funciona si es fa servir el prefix de cada navegador) que s'aplica al text de tipus placeholder dels controls dels formularis.

CSS3 als navegadors

Fins a la versió 9, Internet Explorer no ha aplicat els selectors avançats pertanyents a CSS3. La resta de navegadors els va incorporar molt abans.

No és l'únic cas d'aplicació tardana. Internet Explorer és, sens dubte, el navegador que més triga a incorporar millores de CSS3 que els altres navegadors porten implementant des de fa un temps. Però la resta també triguen a aplicar aquestes millores de CSS (lògic ja que CSS3 no deixa de créixer).

El seu successor, Edge, és més respectuós amb el CSS3. Però la qüestió és que encara hi ha molts usuaris que utilitzen Internet Explorer.

També pot passar que alguns navegadors accepten algunes coses i d'altres no. Per exemple que accepten la pseudoclasse avançada ::selection i tanmateix no reconeguen :not. Això complica enormement la feina del dissenyador de pàgines web.

Les últimes versions de tots els navegadors ja incorporen la majoria dels selectors.

Tot i que de vegades cal comprovar que no es requereix prefix. És el cas del selector ::selection, només els últims navegadors el reconeixen i en el cas del Firefox cal acompanyar-lo del prefix –moz-

Ejemplo:

p::-moz-selection{
    background-color:red;
}

En definitiva, cal fer servir amb cautela aquest tipus de selectors.

A causa de la manca de compatibilitat tan gran d'Internet Explorer amb CSS3, és habitual utilitzar codi concret per a aquest navegador (i que només volem que execute aquest navegador) per a això s'utilitza aquest truc:

<!--[if lt IE 9]>
    ..codi ignorat per tots els navegadors excepte per Internet Explorer anteriors a la versió 9
<![endif]-->

Descarrega l'html i la imatge des d'ací. Després, sense tocar l'html (la part del body) realitza les següents accions.

  1. Canvieu el fons (background-color)de la pàgina a negre i el color del text a blanc.

  4. Si vols canviar el color dels paràgrafs a groc: He de canviar les seccions.
  5. Si vols canviar els colors de la madrastra a verd i violeta…
  6. Posar el css dins de l'html
  7. Treure el css fora de l'html

Objectiu: comprovar que l'ús del CSS ens permet canviar fàcilment les característiques del document sense haver de recórrer-lo. En un document no sembla molta millora, però imagina't en una web amb 100 pàgines…. Canviar el tipus de lletra seria molta feina.
